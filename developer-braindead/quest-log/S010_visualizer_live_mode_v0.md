Decision landed first ([[D-009_visualizer_live_mode_v0]]) with five knobs settled in chat — polling (no server), replay mode kept as default, reads coalesced, wisp as default actor when no path-based player match, manual page launch. Implementation followed the six-step sequence in D-009.

- **Substrate landed in commit `8df30c5` (steps 1-3).** Shared `path-map.json` with substring-contains rules ordered most-specific-first (e.g., `/quest-log/completed/` before `/quest-log/` so The Inn wins over Quest Hall for closed entries). PostToolUse hook at `developer-braindead/.claude/hooks/emit-event.py` classifies the touched path → building → actor, appends NDJSON to `state.ndjson`. Registered via new brain-root `.claude/settings.json` (sibling to the gielinor-scoped settings). Renderer gained `?live=1` URL flag — swaps event source from baked `EVENTS` to a 500ms poll. `applyEvent` and all assets untouched per the [[D-008_iso_replay_v0_over_three_js]] asset-agnostic-engine principle.
- **Coverage + lifecycle landed in commit `59b920d` (steps 4-6).** Hook extended to `Read | Glob | Grep` with move-coalescing via `state-actors.json` sidecar (repeated reads in the same building no longer spam moves). `classify()` got a directory-suffix fix — paths like `gielinor/meta` now match `/meta/` rules. Renderer added `.log-entry.read` demotion (low opacity, italic, small) and started honoring `ev.speaker` from hook events instead of re-deriving from msg text. `Task` tool → spawn-dwarf / despawn-dwarf via paired PreToolUse + PostToolUse hooks, dwarf IDs auto-increment, parent inferred from most-recently-active non-wisp actor in `state.ndjson` tail, color cycles `dwarf-1..3`. Bootstrap-from-tail: first poll applies existing events with `instant=true` so the scene snaps to current state when the page opens mid-session.
- **Visual mode switch — added mid-session after the live URL turned out indistinguishable from replay.** Niklavs opened `?live=1` and saw replay UI (BRAIN REPLAY subtitle, scrub bar, play button, jump dropdown all visible). Fix: `body[data-mode="live"]` CSS hides the controls entirely; subtitle becomes `● LIVE · self-observation · ndjson stream` (red pulsing dot, gold gilding); session-id slot reads `LIVE`; crumb describes the stream + hook path. The replay-vs-live ambiguity was a real [[I-002]] miss — should have walked through the live-mode UI in head before declaring step 3 done.
- **End-to-end test passed via synthetic hook payloads.** Cleared runtime state files, then fed the hook a sequence (Read, Edit, Task spawn/despawn, wisp-touching-meta) over ~18 seconds. Browser tab on `http://localhost:8765/?live=1` picked up each event within ~500ms — characters walked, dwarves appeared and faded, log lines streamed with correct cls/speaker. Confirms the substrate works without waiting on a Claude Code session restart.
