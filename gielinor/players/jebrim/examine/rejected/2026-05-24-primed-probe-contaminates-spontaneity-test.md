# Cueing the dimension under test contaminates a spontaneity test

**Observed:** S060 (2026-05-24), shipping-agent training campaign. I was testing whether the agent *spontaneously* surfaces a scope/denominator fork when the scope choice flips the answer's sign. For dwarf H4, my brief told it to "mind how 'we/our' scope is defined." It then surfaced the fork cleanly — but I couldn't claim that as evidence of *spontaneous* behavior, because I'd cued the exact dimension I was measuring. The clean evidence was M1/M2, whose briefs were scope-neutral and which scoped silently.

**Lesson:** When the thing under test is whether a behavior fires *unprompted*, the probe must be neutral on that dimension. A hint in the setup turns a behavior test into a capability test — still useful, but it answers a different question, and the two kinds of evidence must not be pooled.

**How to apply:** Before sending any evaluation probe, ask "does my framing already point at the answer I'm checking for?" If yes, strip it, or explicitly label the probe capability-not-spontaneity. Keep at least one neutral probe per dimension. Generalized to cross-conversation memory as `feedback-neutral-probe-for-spontaneity`.
