# S276 — 2026-06-19 — Cockpit BRAIN side-panel: feed visibility + canvas remount fix

- Entered dev-brain via "lets develop gielinor". Niklavs reported the right **BRAIN side panel** (Obsidian graph dock + activity feed) broken: on first cockpit open the feed didn't render (only the graph), and after toggling the board feed-button off→on the whole panel went blank. Diagnosed **evidence-first** — headless Chrome (CDP driver over the `websockets` lib) seeded the suspected persisted `localStorage` state, screenshotted, and drove the off→on toggle. Default/fresh state rendered fine, proving the bug was **state-dependent**.
- **BUG A (symptom 1 — "only the map, no feed").** `[[cockpit]]/web/styles.css` `.brain-dock` is a fixed-height `flex:none` (`height: var(--brain-h)`). An oversized persisted `--brain-h` (toward the 820 clamp max, from a past gutter-drag) rendered the dock ~1107px at zoom 1.35 in a ~905px column → feed head + list pushed entirely below the fold. Nothing reserved feed space. **Fix:** `.brain-dock { max-height: calc(100% - 170px) }` — caps the dock so the feed always has room regardless of `--brain-h` or zoom; makes the user's already-saved bad value harmless (no localStorage surgery).
- **BUG B (symptom 2 — "completely gone" after hide→show).** `[[cockpit]]/web/brain.js` `draw()` only calls `applySize()` (which sizes the canvas *backing store*) when the rendered box differs from the module-global `view.w/view.h`. On **remount** those globals still held the prior instance's size = the fresh canvas's size → size-sync skipped → new canvas kept its default **300×150** buffer → all nodes drew off-screen (blank panel; header still showed "690 notes · 1742 links" because `loadGraph` ran fine — only the draw target was wrong). **Fix:** size the backing store to the real box on every (re)mount in `mountBrain`; the loop still self-heals later resizes.
- **Verified** headless against the exact worst-case state (`brain-h=820`): first open → dock caps at 676px with feed visible (head top:685, items 174px, within the 905px viewport); after hide→show → graph renders full-size, not blank. No JS exceptions; `node --check` clean; CSS braces 365/365 balanced. **RUNTIME-UNVERIFIED in the live pywebview window** — the running cockpit holds stale JS; needs a **relaunch** to pick up the fix (standing cockpit-relaunch-verify caveat).
- Build-lesson harvested ([[build-lessons]]): a state-dependent UI bug that won't reproduce from a fresh page is a *persisted-state* bug — seed the real `localStorage`, don't conclude "works" from the default render. Diag probes (temp `_diag*` files + CDP driver) created and swept at close.
