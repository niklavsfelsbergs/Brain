## How to run the cockpit

Launch via the **cockpit / Switchboard** Desktop/Start-menu icon — it starts its own backend (`cockpit/Switchboard.vbs` → `app.py`; a pywebview window + aiohttp backend + PTY driver in one process). Hooks write state to `switchboard/state-*` files + `~/.claude/status/<sid8>.json`; the cockpit backend reads those. A relaunch is needed to pick up backend/JS changes (the running window holds stale code). `node --check` + `py_compile` are the cheap pre-commit gates for any web/hook edit.
