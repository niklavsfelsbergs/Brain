# S009 — 2026-05-21 — Visualizer six-delta aesthetic pass + edge-frame fix

Continuation of [[S008_iso_visualizer_v0]]. The respawn handed off six concrete deltas inspired by a pixel-art reference image Niklavs shared. Niklavs said "do it all" — single-pass execution across all six, then one round of feedback iteration. Engine untouched throughout, per [[S008_iso_visualizer_v0]]'s asset-agnostic-engine principle.

- **Six deltas landed in order.** (1) Always-on ornate `◆ BUILDING NAME ◆` chips replacing hover-only plaques, via a reusable `ornateLabel()` helper. (2) Sky band at viewport top: gradient + three mountain silhouette layers + snow caps + a tree-line silhouette, with the grass diamond's tip poking up into it. (3) Six new ground-detail sprites in `<defs>` (mushroom, tuft, pebble, dandelion, daisy, lavender, mushroom-ring) sprinkled ~3-4× denser than v0 with a 72px building-clearance check. (4) Every building bumped taller — Spellbook Tower now the dominant landmark at `h=156 r=84` with crenellations + finial + bright star; per-building character touches on Meta Town Hall (corner turrets, upper-row arched windows), Lorebook Library (taller ivy + upper windows), Quest Hall (second-floor lit windows). (5) Log panel restyled from `EVENT LOG` to `COMMS` with six speaker tabs (ALL/JEBRIM/ZEZIMA/DWARVES/WISP/COMMITS), each with a colored dot; entries auto-tagged with `data-speaker` via a `deriveSpeaker()` heuristic (regex for dwarves, else `activePlayer || wispActive ? 'wisp' : 'system'`), filtered via CSS `data-filter` attribute. (6) Floating `◆ JEBRIM AT QUEST HALL ◆` focal label tracking the active player, animating with the actor on every move, hidden during unscoped/wisp sessions.
- **Three feedback fixes on the first review pass.** (a) Building labels were inside per-building `<g>` groups, so characters and the vignette rendered on top of them; extracted all labels into a new `<g id="building-labels">` layer drawn last in the SVG, after vignette and characters. (b) Two trees were standing on the pond — iso-coord positions `(8, 4)` and `(9, 0.5)` mapped right onto pond center `(740, 620)`; added `clearOfPond` check (radius 104 for trees, 90 for small details). (c) Trees were scattered through the interior; replaced inner-ring iso-coord scatter with explicit screen-coordinate edge loops (top below horizon, bottom, left, right) so trees frame the picture; bushes/flowers/rocks still scatter through the open map with building, pond, and path clearance.
- **Engine surface preserved.** Timeline + `applyEvent` + RAF + scrub-seek untouched. Added state: `wispActive`, `currentBuilding` per actor, `LABEL_Y_OFFSET` per building. Added functions: `ornateLabel`, `buildLabels`, `updateFocalLabel`, `deriveSpeaker`. Refactored `buildFlora` from iso-ring topology to edge-frame topology. New top-layer SVG group `<g id="building-labels">`.
- **[[I-002]] in practice — partial application.** Mental UI preview before each delta caught most issues (sky band height vs grass tip, building heights vs viewport ceiling, focal label vs name plate collision, speaker-derivation correctness). What it missed: SVG *render order* (labels rendered below characters because they were nested in the same group as the walls) and *placement collisions with non-building features* (trees on pond — only buildings were checked). Both surfaced on first review. Extension worth noting but not yet a new I-NNN: mental preview must include z-order and all collision targets, not just the static visual.
