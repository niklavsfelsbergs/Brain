# `spellbook/` — about

Procedural memory for the dev brain. Dev rituals — things to *do*, not just *read*. Each spell is a procedure invoked at a known moment.

**What lives here.**
- `respawn-ritual.md` — session-start procedure.
- `session-close.md` — session-end procedure.
- `entry-formats.md` — reference for how to write decisions, assumptions, open questions, risks, identity entries, and quest-log entries.

**Distinct from main brain's `spellbook/`.** The main brain's spellbook holds the *agent's* skills (teleports, rituals it invokes in normal operation). This dev-brain spellbook holds *our* dev-session procedures. Different audience.

**Conventions.**
- Spells are short. Five steps is a lot; ten is too many.
- Each spell states *when invoked* and *what it produces*.
- Spells can reference each other (`session-close.md` invokes the quest-naming step from `entry-formats.md`, etc.).
