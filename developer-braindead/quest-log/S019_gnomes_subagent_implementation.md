# S019 — 2026-05-21 — Gnomes sub-agent implementation

- Designed and shipped **gnomes** — a third sub-agent role (principal/dwarf/gnome) for structural housekeeping rituals. Founding decision in [[D-016]]; covers session-close, per-player alching, drafts-triage. System-namespace (one agent config; spawn brief carries player scope as parameter). Heuristic auto-spawn keyed off turn count, players touched, and pending-drafts count; light rituals stay principal-self so the procedure doesn't drift.
- Architectural enforcement landed via two hooks. New `gielinor/.claude/hooks/gnome-write-boundary.py` gates the gnome write surface (drafts/proposals/inventory/quest-log across players, plus globals' drafts and `players/inbox/`) and blocks `confirmed/`, `lorebook/decisions/`, `keepsake/current.md`, `meta/`, `spellbook/rituals/`, body files, and `.claude/`-infrastructure. `block-sub-dwarf-spawn.py` renamed to `block-sub-spawn.py` and generalized to fire on either `CLAUDE_BRAIN_DWARF=1` or `CLAUDE_BRAIN_GNOME=1`. Settings wired in `gielinor/.claude/settings.json`.
- Operating spec landed at `gielinor/spellbook/skills/spawning-gnomes.md` — single source of truth for the spawn heuristic numbers (session-close: >15 turns / ≥2 players / >5 drafts; alching: >20 turns since alch / >10 drafts / never-alched + day-1+; drafts-triage: >10 drafts brain-wide). Rituals reference this file rather than carrying copies. Includes briefing template, write surface, discipline rules, reporting format.
- Agent config at `gielinor/.claude/agents/gnome.md` (brain root, so brain-root dev-brain sessions can spawn). Tools: Read/Edit/Write/Glob/Grep/Bash. No Agent/Task — gnomes can't spawn sub-agents (hook-enforced).
- Rulebook updated across three files: `gielinor/meta/modes.md` renamed the axis from "principal vs dwarf" to "principal vs sub-agent" and added the Gnome role section + write-allows/blocks; `gielinor/meta/write-rules.md` added Session-close and Drafts-triage rows to the ritual write-reach table and added the gnome boundary to the hooks-enforced list; `gielinor/CLAUDE.md` 4 → 5 architectural guarantees, with #4 = gnome write boundary and #5 = no sub-spawn. Both rituals (`close-session.md`, `alching.md`) got a step 0 spawn-decision that references `spellbook/skills/gnomes.md` for the heuristic.
- Visualizer integration (sprite, workshop building, spawn/despawn events on agent invocation) deferred per [[D-016]] *Things deferred*. Next session: principal approval + visualizer implementation.
