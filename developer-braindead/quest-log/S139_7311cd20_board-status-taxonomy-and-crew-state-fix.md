# S139 — 2026-05-31 — Cockpit board status taxonomy (main + sub-bubbles) + crew-state fix

Continuation of the same dev session that closed [[S134_7311cd20_cockpit-board-ux-sort-highlight-ritual-chips|S134]] (re-opened post-CLOSING on two new principal reports). Two stacked board fixes, both in the backend reader (`build_session_model`) with board/CSS rendering — a rendering-layer remap, the hook contract untouched.

- **Crew-state bug (the first report: "YOUR MOVE shows when actually waiting for subagents").** Root cause: `EVENT_STATE["Stop"] = "your_move"`, and the sidecar's subagent→busy logic only runs on the Task `Pre`/`PostToolUse` events — so a `Stop` firing while foreground crew is still out stamps `your_move`, and the `crew` tag only attaches to `busy`. Fixed with a backend invariant: a row with foreground crew still pending can't read ball-in-your-court → `if state == "your_move" and pending: state = "busy"`. Self-heals every 2s poll (re-derives to `your_move` once the last subagent returns); trusts the same pending list the crew chips already draw from. Caveat documented (orphaned-trace risk shared with the chip; `tools/reap.py` + the stall backstop).
- **Status taxonomy (the second report: "idle/stalled should be a side status, not their own").** Restructured the board into a small set of **MAIN** statuses (the primary chip) + **SUB-bubbles** (secondary). MAIN = `BUSY` / `YOUR MOVE` / `ACTION NEEDED` (was NEEDS YOU) / `WRAPPING UP` (was DONE) / `ALCHING` / `BANKSTANDING`. SUBs = idle · stalled · consulting · drafts · a demoted ritual; crew rides its own kind-letter row. **idle/stalled no longer relabel the chip** — a quiet row keeps its real main status and gains a sub-bubble (reverting [[S134_7311cd20_cockpit-board-ux-sort-highlight-ritual-chips|S134]]'s your_move→idle / busy→stalled *relabel*; the liveness logic stays, it just emits a sub now). **Rituals promote to a main chip, but a ball-state wins** (principal multiple-choice): `ALCHING` while churning → `YOUR MOVE · alching` when parked → `ACTION NEEDED · alching` on a question.
- **Implementation.** `cockpit/backend.py` — `MAIN_RANK`/`IDLE_SINK_RANK`; per-session derive `main` + `subs` (ball-state-wins precedence), liveness sub-flags, greying only on idle, rank from main with idle sunk; emits `main`/`subs` (kept `state` for compat + ping transitions, `tags` aliased to `subs`). `cockpit/web/board.js` — `MAIN_LABEL`/`SUB_LABEL`, chip + row-class off `s.main`, sub-bubbles off `s.subs`. `cockpit/web/main.js` — cockpit-own rows emit `main`/`subs`. `cockpit/web/styles.css` — `.state-alching`/`.state-bankstanding` chips + `.flavor-idle`/`.flavor-stalled` sub pills.
