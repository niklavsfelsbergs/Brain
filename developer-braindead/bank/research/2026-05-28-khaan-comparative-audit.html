What Khaan is

Same raw materials as us — a markdown brain, Claude Code, subscription (no API / no headless), hooks as the substrate. A fundamentally different philosophy on top.

Kha'an

single autonomous entity · EC2 (goodhealth.lv) · builder: Justs Ceplis

  • Neuroscience memory. cortex/ regions — amygdala (trust/emotion), broca (voice), episodic, identity, prefrontal (rules), procedural, semantic, fusiform (people). Consolidation levels L1→L4.
  • Hooks as positive enforcement. Every turn blocks all tools except the one clearing the current gate: recall → situational-identity → decision, in order, before the agent may act.
  • Real retrieval. recall-engine scores + reranks cortex against the prompt (inverted index, domain dictionary, bundles) into active-recall.json each turn.
  • Autonomous producer. autonomy-engine walks dashboard tasks through skill chains; learning-engine runs a 10-phase consolidation cycle on a counter trigger.
  • Engineering rigor. every skill ships verification/check.py + golden files; create-skill enforces a strict four-folder pattern.

Gielinor

multi-actor cognitive system · local + cockpit · principal: Niklavs

  • RuneScape memory. per-player bank/ · quest-log/ · examine/ · keepsake/ plus globals (niksis8/ · lorebook/ · meta/). Drafts → confirmed pipeline.
  • Hooks as negative enforcement. block writes to confirmed/, block deletes, block sub-spawn, per-role write boundaries. The positive loop is ritual discipline.
  • Agentic search. grep + an Obsidian-navigable wiki-link graph. Scored recall (GraphRAG, §N) designed but parked.
  • Human-in-the-loop. the principal is the producer; rituals (respawn / alching / bankstanding) tend the brain. Scheduled autonomy (§C) deferred.
  • Observability-first. the cockpit (PTY + board/console/feed) with a regression harness for that surface.

What's similar — the convergent design

Two people built personal Claude-Code brains independently and landed on a startling amount of the same scaffolding.

  1. File-based markdown brain over a subscription harness, hooks load-bearing. Both reject API / headless metering and drive everything through Claude Code hooks over markdown + JSON on disk.
  2. Layered memory with ID'd atomic entries.
    AM### / BR### / E### / ID### / PR### / R###  ≈  D-NNN / S-NNN / B-NNN — prefix-stable IDs, descriptive filenames.
  3. Raw observation → consolidated knowledge as a pipeline.
    L1 → L4 promotion  ≈  drafts/ → confirmed/ — "captured" separated from "canonical."
  4. Identity composed and surfaced at boot.
    core-identity → boot-identity.md kernel  ≈  respawn.md + examine/ + keepsake/ — "who am I this session" is recomposed, not static.
  5. A relationship-with-the-human layer.
    fusiform / identity "Who Is Justs" + amygdala trust  ≈  niksis8/ + niksis8_character/
  6. A learned voice layer that decodes the human's shorthand. Khaan's broca ("scrap"/"sooo"/"sataisi"/"goodboi") ≈ our persona.md + the Niklavs voice profile. Both even land the same anti-slop rule — Khaan: "do not append summary blocks, the diff is visible"; us: "don't summarize what just happened, the principal can read the diff."
  7. Archive discipline + git as the audit log. Both keep an archive/ mirror and commit the brain to git as durable history.
  8. Skills/rituals as procedural memory, and an observability surface. Their dashboard/*.json digests ≈ our cockpit. Both wanted to watch the brain run.

The headline: the markdown-brain + Claude-Code-hooks + subscription + boot-time identity + ID'd atomic memory + draft-to-canonical pipeline pattern was arrived at twice, independently. Strong signal it's the right shape.

Gielinor learns  What Gielinor can learn from Khaan

Our hooks only say "no." Theirs say "now do this." Every guarantee we enforce is negative; the positive loop (respawn properly, ground before advising, alch on cadence, post an OPEN) is left to prompt-discipline — and our own audits keep finding it leaks. Khaan gate-enforces its cognitive loop: you physically cannot call a tool until recall + identity + decision have refreshed.

  • Mechanical grounding gate

    Our memory literally carries "grounding before advice" and "check own memory before working repo" as repeated feedback misses. Khaan would make those non-skippable with a per-turn gate + escape clause. We could gate "read keepsake/current.md before substantive player advice" the same way.

  • A scored recall engine

    We keep parking GraphRAG (§N) behind §C. Khaan shipped retrieval — inverted index + domain dictionary + bundles, reranked every turn, deterministic (no LLM, no billing). Our brain "is agentic search," but has no scored surface that pre-loads relevant memory before the turn. Their recall-engine is a working blueprint.

  • The autonomous producer we keep naming but not building

    Our D-027 inward/outward gap and the §C pilot are Khaan's autonomy-engine + consolidation cycle: a scheduled loop that acts on the queue instead of waiting for the principal. They're at cycle #116; we're "all inbox, no producer." Their orchestration.yaml (trigger → skill-chain) is a clean, copyable pattern.

  • Golden-file verification per skill

    Every Khaan skill ships verification/check.py + golden_run.md. We have test_backend.py for the cockpit and nothing for rituals/skills — yet "verify enforcement fires" is one of our hardest-won lessons. Bake a golden harness into the skill template.

Khaan learns  What Khaan can learn from Gielinor

Their brain auto-rewrites its own identity with no human gate — and the identity it's writing is mostly trust-collapse. learning-engine auto-promotes self-observations into cortex (including the identity region) on salience thresholds, then recomposes the boot kernel from them. The current boot-identity.md is dominated by fabrication charges and demotions to "USELESS." An ungated self-observation → identity loop can entrench exactly that drift.

  • HITL identity gating

    Our confirmed/ is hook-enforced user-only; identity changes go drafts → (principal approves) → confirmed. Khaan would benefit from a draft-gate on at least the identity/amygdala regions — let the cycle propose, let the human ratify. (Their create-skill is gated this way; the memory isn't.)

  • A separate rationale layer (the "why")

    Khaan's create-skill explicitly bans rationale from SKILL.md — efficient for the reading LLM, but then the why lives nowhere, and a future cycle can't tell a load-bearing rule from an arbitrary one. Our lorebook/ (D-NNN decisions, anchored and linked) + the whole dev brain are that store, kept out of the operating path. Keep the why — somewhere navigable.

  • Multi-actor separation with bounded write surfaces

    Khaan is one monolithic self; agents/jeff and agents/teacher are empty placeholders. For a builder "juggling eight threads," our model — domain-scoped players, system actors, per-role write-boundary hooks, explicit delegation — contains blast radius and keeps context clean. Their monolith piles work-context, identity, and emotional state together.

  • The two-brain split

    We separate the cognitive system (gielinor/) from its construction log (developer-braindead/). Khaan builds itself inside itself. Splitting them keeps the operating brain uncluttered and gives the build its own discipline.

  • Never-delete as architecture

    We hook-block deletes and mirror to archive/. Khaan's consolidation actually loses memory — the hippocampus queue caps at 50 and drops the lowest-salience entry; the hygiene phase decays high-miss entries. Unrecoverable. A salience-0 memory today can be the load-bearing anchor next month.

The finding — complementary halves

Each brain has built the thing the other only talks about.

Khaan shipped

The autonomous producer + scored retrieval engine that gielinor keeps deferring (§C / §N).

Gielinor shipped

The human-gated identity discipline + rationale layer + multi-actor isolation that Khaan's monolith lacks.

Put bluntly: Khaan is a running cognitive loop in search of a brake; gielinor is a well-governed brain in search of an engine. The two are almost the same system viewed from opposite ends.
