# S028 — 2026-05-22 — Subtask channel and Guthix

- Picked up [[subtask-channel-design]] (Step 2 from S027 respawn). Goal: a third visualizer channel between **intent** (agent voice, slow, scope) and **action** (hook voice, fast, raw) — a hook-authored natural-language micro-step that updates the bubble at medium cadence so the sprite feels alive while the agent is hammering tool calls under a stale intent. The principal's S027 close complaint — *"sprite gets stuck on one intent for 10 minutes while doing a billion things underneath"* — is the load-bearing motivation.
- **Slice planned.** Ship in four steps: (1) hook synthesis dict + `subtask` event emission, JS dispatch is a no-op pre-step; (2) visualizer reception — `actorSubtask[actorKey]` state + dispatch case; (3) bubble rendering extension (Option A — secondary muted line under intent); (4) debounce + live test. Step 1 first.
- **Divergence from design doc.** Doc puts debouncer in the hook ("in-memory dict per actor"). But the hook is a fresh process per tool call — no persistent in-memory state. Debouncer moves to the visualizer (long-lived JS), where in-memory is real. Hook emits every call; visualizer collapses. Bonus: full-fidelity event log survives for replay; debounce can be tuned without re-running anything.
- **Slice 1 — hook synthesis + emission (shipped).** Added `subtask_for(tool_name, tool_input)`, `bash_subtask(cmd)`, `BASH_SUBTASK_TABLE`, and `emit_subtask(wall, actor, tool_name, tool_input)` to `developer-braindead/.claude/hooks/emit-event.py`. Wired emission into all four sites that already attribute an actor: `handle_bash`, `handle_write_or_read` sub-agent path, `handle_write_or_read` main-thread path, and `handle_task_pre`. Event shape: `{type: "subtask", actor, text, wallTime, source: "hook"}`. SUBTASK_MAX_LEN = 80. Smoke test (`experiments/subtask_smoketest.py`) confirms phrases land cleanly across Read/Edit/Write/Glob/Grep/Bash (with verb-table coverage for git status|diff|log|show|blame / git add|commit|push|pull|fetch / ls family / cat family / python|node|npm|… / echo for sidecar / fallback) / Task / WebFetch / WebSearch.
- **Slice 2+3 — visualizer reception + render extension (shipped).** Added `setSubtask(actor, text)` helper; added `subtask` field on `intents[actorKey]` entry creation; added `case 'subtask'` to `applyEvent`'s switch. Modified `renderIntent` to layer subtask as a styled secondary line under the intent — rendered as `fill="rgba(31,21,12,0.58)" font-style="italic"` to read muted. Subtask occupies line 2 **only** when the intent wraps to a single line (Option A from the design doc; intent wins ties because it carries scope). Subtask is dropped when there's no intent — no bubble-creator role; it's a secondary signal.
- **Deviation from design doc — COMMS append.** Doc recommends subtask events also append to COMMS with a new `.log-entry.subtask` class. Held off: action lines already cover that surface (every tool call already gets a chat line), and adding subtask would near-duplicate. Bubble-only for now; promote to COMMS in slice 4 if the bubble alone doesn't satisfy.
- **Replay demo deferred.** The EVENTS array predates D-014 — no `intent` or `action` events in it yet, only `log`/`move`/`spawn-dwarf`/`commit`/`session-start`. A clean replay demo of the channel would need restructuring a session arc to use intent + subtask + action together. Cheaper to verify live; defer the replay arc to a follow-up.
- **Debounce — slice 4 (open).** Without debounce, every tool call updates the bubble. For a typical turn with ~5–10 calls spread over 30s, that's 3–6s between updates — not strobing. The strobe risk is during fast loops (parallel Read calls, etc). Starting numbers from the design: 500ms min update, 1.5s aggregation, 2s reset gap. Decision deferred to live test — may not be needed at all.
- **Pivot mid-session — bankstanding wisp confusion → Guthix.** Live-testing the subtask channel surfaced a UX issue: principal saw Jebrim's session running bankstanding and asked *"why is my Jebrim walking around as a wisp?"* Tracing it: the session wrote intent to `wisp.txt` (correct per the old `communication-protocol.md` — bankstanding = system voice), and subtask events made the wisp's activity feel intensely Jebrim-shaped ("reading write-rules.md", "reading modes.md", etc) — exposing an architecture/intuition mismatch the old design papered over. S028 expanded scope to fix it.
- **Decision: introduce Guthix as the bankstanding deity.** Wisp was carrying two unrelated meanings (unscoped session + system curation) and the second one didn't fit. Guthix is a new dedicated actor for bankstanding: hooded green-robed sprite (lore-aligned — RuneScape god of balance), floats above buildings instead of standing, glides between them without a walk cycle. Wisp shrinks back to "unscoped session" only. Player sprites stay in place while Guthix works the brain. Principal greenlit "Yes Guthix. Also make him look like Guthix and make him float around instead of sit like the others."
- **Shipped (Guthix end-to-end).**
  - **Visualizer.** New CSS palette (`--guthix-robe` family + `--guthix-text/dot/swatch`), float keyframes (`guthix-float`, 3800ms, ±6px) and glow filter (`guthixGlow`), full pixel-art sprite (hooded green robe, white beard, wooden staff with green gem, soft aura), `ACTOR_Y_OFFSET['guthix'] = -48` so he floats above buildings on every move, `NON_WALKING_ACTORS = {'guthix'}` suppresses `.walking` class + dust trail in `moveActor`, chat lines now say "drifts to" instead of "walks to" for non-walkers. `spawnGuthix` / `despawnGuthix` parallel to wisp/braindead; default spawn location `lorebook-library`. `ensureActorExists` adds the guthix branch (self-heal on first move/intent). `applyEvent` dispatches `spawn-guthix` / `despawn-guthix`. `resetWorld` cleans him up. `deriveSpeaker` puts him at the top of the precedence ladder (above braindead and wisp — bankstanding is the top-level ritual). Legend row + COMMS tab + filter + speaker bullet + chat `.user` color all wired.
  - **Hook.** `GUTHIX_ACTOR = "guthix"`, `GUTHIX_DEFAULT_BUILDING = "lorebook-library"`. New `is_guthix_session()` parallel to `is_dev_brain_session()` — reads `_guthix_session_id` from `state-actors.json` and gates on `_SESSION_ID`. New `_handle_guthix_lifecycle(actor)` called from `handle_intent_write` before the intent event lands: when this session writes to `guthix.txt` for the first time it emits `spawn-guthix` + a log line "Guthix descends to tend the brain" and stamps `_guthix_session_id`; when intent flips to a non-guthix actor it emits `despawn-guthix` + "Guthix recedes — the brain is tended" and clears the marker. `handle_session_end` does the same despawn if the closing session owned guthix. The wisp-fallback in `handle_write_or_read` main-thread path, `current_main_actor`, and `infer_dwarf_parent` all now check `is_guthix_session()` between the dev-brain check and the recency walk so unclassified paths route to Guthix when bankstanding is active.
  - **Meta docs.** New `gielinor/meta/guthix.md` (persona, voice, write reach, visualizer notes). `modes.md` bankstanding section rewritten: voice is now Guthix, intent file is `guthix.txt`, with cross-references to the new file. `communication-protocol.md` active-actor-by-mode bullet adds the bankstanding → `guthix.txt` route. `write-rules.md` adds a "Voice per ritual" paragraph below the Ritual write-reach table — bankstanding = Guthix voice, alching = active player voice, others carry the active actor's voice.
- **Visualizer-side caveats / follow-ups.**
  - Replay-mode demo deferred — same reason as subtask. EVENTS array doesn't have intent/action events; adding a bankstanding arc would mean restructuring.
  - The hook spawns guthix on the *first* `guthix.txt` write. If a session writes to guthix.txt and then never to anything else (and never ends cleanly), the sprite stays until the visualizer's 5-minute idle despawn fires. Acceptable for v1.
  - `is_guthix_session()` is gated on `_SESSION_ID` like `is_dev_brain_session()` — parallel sessions sharing the state file can't fire each other's guthix override.
- **`Hey Guthix` invocation pattern.** Principal asked: *"shouldn't we just have a Hey Guthix option, and it prompts what it could do for me?"* — extended the address-by-invocation router to recognize Guthix alongside players + unscoped. `gielinor/CLAUDE.md` adds `Hey Guthix, ...` row + matching rules updated ("known address" covers players + `unscoped` + `Guthix`). Mid-session switching extends to Guthix (same mini-respawn pattern; Guthix has no `quest-log/` so the bankstanding output is itself the trace). `gielinor/meta/guthix.md` adds a full *Invocation contract* section with a four-item menu — Bankstand / Triage drafts / Survey across players / Audit a global layer — plus a *What he refuses* subsection (redirects per-player work back to alching, dev-brain construction back to dev-brain mode) and a *Returning* section. `modes.md` bankstanding `Set by` line now lists both entry routes (`Hey Guthix` and `let's bankstand`).
- **Live-test debugging (post-ship).** Principal opened the visualizer mid-S028 and reported (a) stale bugged-out sprites cluttering the map, (b) Braindead getting no speech bubbles, (c) a Jebrim session "currently acting but no bubble or movement." Diagnosed and fixed in three small batches:
  - **Map wipe.** Truncated `state.ndjson` + reset `state-actors.json` / `state-dwarves.json` / `state-gnomes.json` / `state-instances.json` via a single Bash python script. Visualizer's pollLive detected truncation (B5 from S027) and `resetWorld()`. Re-stamped `active-mode.txt` so the dev-brain session re-spawned Braindead.
  - **Braindead bubble routing.** Hook's `handle_intent_write` only stripped the `-<sid8>` suffix when the prefix matched `PLAYER_ACTORS`. So `braindead-ed610cbe.txt` emitted `actor="braindead-ed610cbe"` (not `"braindead"`) — visualizer couldn't route to the sprite, bubble never rendered. Added `NON_PLAYER_SUFFIX_ACTORS = {"braindead", "guthix", "wisp"}` and extended the suffix-strip check. Bug has been silently present since D-018 chunk 3 (whenever the agent first wrote per-session braindead files).
  - **Disk-fallback for session→actor recovery.** State wipe surfaced a separate issue: a Jebrim session whose state.ndjson events got wiped attributed subsequent Bash to `wisp` because the recency walk for that session found nothing. Added `_session_actor_from_disk()` that checks `.claude/intent/<actor>-<sid8>.txt` on disk — the intent file outlives state.ndjson, so it survives truncation/reset cycles. Plugged into `current_main_actor()`, `infer_dwarf_parent()`, and the wisp-fallback in `handle_write_or_read` main-thread path. Helps sessions using per-session intent files; sessions on bare files (the in-flight Jebrim session 58f8e88a, writing to bare `jebrim.txt`) recover on their next respawn.
- **Protocol tightened.** `gielinor/meta/communication-protocol.md` intent-narration paragraph rewritten: per-session intent filename (`<actor>-<sid8>.txt`) is now the only sanctioned shape, for every actor (players + Braindead + Guthix + Wisp). The bare-file fallback is removed — if `CLAUDE_CODE_SESSION_ID` is unavailable, surface to the principal rather than silently fall back, because bare files have no session ownership and break the recovery path. Resolves the recurring "in-flight sessions clobber each other" issue ([[S014]], [[S022]], [[S023]], [[S027]] all touched this).
