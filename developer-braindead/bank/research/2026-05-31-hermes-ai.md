# Hermes Agent (Nous Research) — the closest sibling to gielinor in the wild

> **What this is.** External recon captured [[S138_81f1509c_hermes-ai-research]] (2026-05-31, dev-brain, sid 81f1509c) when the principal asked "what is Hermes AI." It's an open-source autonomous agent built on the *same architectural family* as gielinor — persistent memory, self-authored skills, subagents, MCP, local-first. The most useful external reference point yet for **positioning** the brain and for **build ideas**. Sources at the foot.

## Name disambiguation

"Hermes" is overloaded in the AI space. Three distinct things:

1. **Hermes Agent** — Nous Research's open-source autonomous agent, released **Feb 2026**. ← *this is the "new thing."*
2. **Hermes (LLM series)** — Nous Research's older line of fine-tuned open-weight models (Hermes 2/3/4). Not the topic, but same vendor — the agent can run on these.
3. **`get-hermes.ai`** and assorted unrelated startups carrying the name. Noise.

This note is about #1.

## What Hermes Agent is

An autonomous AI agent designed to run on **your own infrastructure** (a $5 VPS up to GPU clusters / serverless like Modal & Daytona), MIT-licensed, Python-first (~89%, TS frontend). One-line installer (Linux/macOS/WSL2; Windows in beta). Tagline: *"the agent that grows with you."*

### The four primitives that matter to us

- **Persistent memory across sessions.** Agent-*curated* memory (it decides what to keep, prompted by periodic "nudges"), FTS5 full-text search over past sessions with LLM summarization, and explicit user-modeling. "Remembers everything across restarts."
- **Self-authored, self-improving skills.** After a complex task or feedback, it *writes its own skill file* and refines it on later use. Compatible with the `agentskills.io` standard.
- **Subagents.** Spawns isolated sub-agents for parallel workstreams.
- **40+ tools + MCP.** Browser automation (navigate/click/type/screenshot), vision, sandboxed code execution, image-gen, TTS; extensible via MCP servers. Six terminal backends (local, Docker, SSH, Singularity, Modal, Daytona).

### Reach & cadence

- Multi-platform front door: CLI, Telegram, Discord, Slack, WhatsApp, Signal.
- Claimed #1 agent on OpenRouter by usage (~224B daily tokens by May 2026). Very large, fast-growing star count — **figure unreliable**: SEO sources range 46k/140k/174k; GitHub itself read ~174k at fetch. Treat as "huge and growing," not exact.
- Releases move weekly through 2026: v0.7 *Resilience* (pluggable memory providers, credential rotation, anti-detection browser) → v0.8 *Intelligence* (background-task auto-notifications, live model-switching) → v0.13 *Tenacity* (a **durable multi-agent Kanban board** — heartbeat, reclaim, **zombie detection**, auto-block on incomplete exit, per-task retries, hallucination recovery).

| front doors | CLI + 5 chat platforms | cockpit PTY (single operator) |

### Where they diverge (the real positioning)

Same architectural *family*, opposite branch on two axes:

- **Generic product vs bespoke self.** Hermes is a tool *anyone* installs and points at *their* life. gielinor is **one** cognitive system with a fixed identity, a self-model, and a named cast. Hermes has users; gielinor has *Niklavs*.
- **Automation vs gated identity.** Hermes self-improves *freely* — it curates its own memory and writes its own skills with no human in the loop. gielinor's defining choice is the **opposite**: identity, decisions, and per-player bank are **draft-then-approve**; the principal signs off on what becomes canonical ([[D-027_inward_outward_build_imbalance]] context; the whole `write-rules` gate philosophy). Hermes optimizes for autonomous capability; gielinor optimizes for *trustworthy, principal-shaped* capability.

That contrast is the one-liner for explaining the brain to people: *"It's the same primitives as Hermes Agent — memory, self-authored skills, subagents — but built as a single human-in-the-loop self rather than a ship-it-to-everyone product."*

## What we could learn from it

See the chat synthesis ([[S138_81f1509c_hermes-ai-research]]) for the ranked version. Headlines:

1. **Durable multi-agent coordination (their v0.13).** Heartbeat / reclaim / zombie-detection / auto-block-on-incomplete-exit / per-task retries — this is a *productized* version of what our cockpit + comms + reaper + graduation do ad-hoc. The [[S131_63750f50_construction-side-severity-audit|S131]] audit's "orphaned dwarf traces nobody reaps" finding is exactly a zombie-detection gap. Their Kanban is a concrete reference design.
2. **Agent-curated memory with periodic nudges.** They automate the "what's worth keeping" decision with scheduled self-prompts. Ours is principal-gated by design — but the *nudge* mechanic (a periodic "anything to consolidate?" trigger) is the autonomy half we've parked (§C scheduled trigger). Their model shows the harvest pump running on a timer, not just at session-close.
3. **`agentskills.io` skill standard.** A real interop standard for portable skills exists. If we ever want `spellbook/skills/` to be portable (or to *import* community skills), there's a format to target rather than invent.
4. **Front-door breadth.** Telegram/Discord/Slack/WhatsApp/Signal as first-class gateways. Relevant to the parked mobile-access question (sibling 5b18b6f6) — the answer "expose the agent over a chat platform" is what Hermes already does.
5. **Substrate proof.** A $5-VPS-to-serverless agent with persistent local memory is *running in production at scale* — de-risks our parked ascension/VPS path. The substrate isn't speculative anymore.

### What NOT to copy

- **Free self-modification.** Their auto-write-your-own-skills with no gate is precisely the thing our draft/approve discipline exists to prevent. Don't let "Hermes does it automatically" erode the HITL gate — that gate *is* the product here.
- **Generality.** Resist re-theming gielinor into a generic installable (the product-shape discussion, sibling 234a9b87). Hermes already owns the generic-agent slot; gielinor's edge is being *one specific deep self*, not a framework.
