# S075 — 2026-05-24 — Agent prose into the cockpit FEED (kind:"say")

- **The ask.** Principal pointed at the agent's visible terminal prose — the running commentary it writes *between* tool calls (the Understanding/Plan preamble, "the cockpit tree is clean…", "a sibling just wrote…") — and asked why those can't show in the cockpit FEED, not just the curated PROGRESS intent line. *"This is what I would want."*
- **Why it was absent (the real answer).** The FEED is hook-built (`chat.ndjson` + comms mirrors, merged by `api_feed`). Claude Code hooks **never receive the assistant's message text** — they fire on tool calls / prompts / Stop, none of which carry prose. So the FEED structurally couldn't see it; the whole intent-narration mechanism exists to paper over exactly that gap. But the prose is on disk: every assistant `text` block lands in the session transcript `.jsonl`, the same source `backend.py::parse_transcript` already reads for `/history`.
- **The fix (transcript-tail, principal-chosen over live-stream capture so it covers VS Code sessions too).** [[status-sidecar.py]] now tails each session's transcript from a persisted byte offset (`record["say_offset"]`), emitting each *new* principal-authored assistant `text` block to `chat.ndjson` as a new `kind:"say"`. Skips `thinking` blocks and sub-agent (`isSidechain`) turns. First fire baselines the offset at the current transcript end → a resumed session never replays its backlog. Trailing partial line held until complete; read errors leave the offset untouched (no double-emit, no loss). New `_emit_chat_say` / `_transcript_path` / `_emit_says_from_transcript` + `PROJECTS_DIR` / `SAY_TEXT_MAX=600` constants.
- **No `backend.py` touch** — a new `chat.ndjson` kind flows through `api_feed` automatically (`_ndjson_tail` passes `kind` through). That kept the whole change clear of three live siblings (S074 `2dd85cde` in backend/board/main/fleet, S073 `d82c4fbc` in styles.css, `322cb5c3` in backend/names). [[feed.js]] renders `kind:"say"` clean (no chip) + a **`prose` toggle** mirroring `actions` (default-on, persisted) — the principal's "keep both intent + prose" choice.
- **Verification.** `status-sidecar.py` AST-clean, `feed.js` `node --check` clean. Unit-tested `_emit_says_from_transcript` against a synthetic transcript: emits the two complete prose blocks, **skips thinking + sidechain**, holds the partial trailing line, offset advances exactly to EOF on the next pass.
- **Open.** Principal relaunch + eyeball (running cockpit has stale `feed.js` until relaunch; hook hot-reloads so new sessions emit `say` immediately). Optional `.feed-item.k-say` CSS polish handed to `d82c4fbc` via comms — feature works without it (inherits `.feed-item`). Tuning: `SAY_TEXT_MAX=600` per-block cap may want adjusting once seen live; consider whether the Understanding/Plan preamble doubling the PROGRESS line is noise once prose is on.
