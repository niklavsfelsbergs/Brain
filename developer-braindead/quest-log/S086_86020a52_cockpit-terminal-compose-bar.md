Dev-brain via "lets develop gielinor". Principal's ask: in the cockpit PTY terminal, keep the input box fixed at the bottom while the chat history scrolls — *"otherwise i cant scroll and write at the same time."*

- **Root framing.** A PTY couples input to the scroll grid: claude's input box is the bottom rows of the same xterm buffer as the output, so scrolling up to read history scrolls the prompt away. The fixed-composer UX already existed — but only on the headless `Console` (`cockpit/web/console.js`), which is the *metered* billing path; the subscription PTY terminal (`term.js`) deliberately gave it up. Offered a 3-choice (compose-bar / live-forwarding / scroll-only tweak) with compose-bar recommended; principal picked it.
- **Built (the compose-bar).** A fixed `<textarea>` below the xterm that pipes to the PTY. Two new `TermConn` methods in [[term.js]]: `submitComposed` (`/rename` parity, single-line direct send / multi-line via bracketed paste, re-pin to bottom on send) and `sendEsc`; plus a new exported `TermComposer` component (Enter sends, Shift+Enter newline, Esc cancels, autogrow). `main.js` renders it as the 3rd flex child of `.term-col` under `<Term/>`; `styles.css` `.term-composer` is `zoom`ed like the column header (it's plain DOM — must stay off xterm's unscaled subtree). Board tag `b85.1`→`b86.3`.
- **Merge bug — principal caught it on first live use.** Text half-typed directly into claude's in-terminal box (the terminal is focused by default) survives Esc and gets *prepended* to the next composed send → a merged message (`dssd` + next prompt). Fixed by clearing the `_linebuf` mirror term.js already tracks for `/rename` — backspace the leftover away on Esc (terminal keydown), in `sendEsc`, and at the top of `submitComposed`. Esc still flows through to claude to interrupt. Then two follow-ups: removed the placeholder text, made the bar 3× taller (min-height 38→116px, autogrow cap 140→280px). A third round: Esc in the bar now also clears the textarea's own value.
