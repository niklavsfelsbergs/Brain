# Complete the cheap grounding precondition before substantive output

**Cross-player pattern ([[B-007_2026-05-25_seventh-bankstanding|B-007]] synthesis, first N=2 cross-read).** The same failure mode is confirmed independently in two players: the agent produces confident substantive output *before* completing a cheap precondition, and pays a re-do.

- **Jebrim — `grounding-before-advice` ([[S045_91ee1383_shipping-agent-chart-system-fixes|S045]]).** Dove into chart-fix advice for three turns before reading keepsake; the advice was directionally fine but unanchored (the shipping-agent's location and existing rules sat unread in his own keepsake). Principal caught it twice (*"are you jebrim?"*, *"why dont you know where shipping agent lives"*). Cost: three turns re-anchored.
- **Zezima — `anchor-referenced-subject-before-assuming-its-new` ([[S066_e2362ea0_domajamas-dalas-reconsider|S066]]).** Analyzed a property as a "new, third" one before confirming identity; it was [[S056_e5df54a2_apartment-domajamas-dalas-research|S056]] apartment #1. Continuation cues (*"this property some more,"* *"afterall"*) signalled a known subject; mismatching surface details were let to override that. Cost: a full wasted response. (Zezima's own draft named itself *"Sibling to grounding-before-advice"* — the agent already saw the link.)

**The unified miss.** Two faces of one precondition: **read the active context** (Jebrim) and **correctly identify the referent** (Zezima). Both are cheap; skipping either yields output that is confident, plausible, and wrong-anchored.

**Rule.** Before substantive output in any session, complete the grounding precondition: (a) read the active context (keepsake/current.md at minimum for a player); (b) map continuation cues to existing work and confirm the referent's identity before analyzing on a novelty assumption. The Understanding/Plan preamble confirms the *ask*, not the prior context or the referent — it is not a substitute for grounding.
