# Answer a bundled direct question as its own foregrounded beat

**2026-06-09 · [[S186_91513c92_nfe-domain-deepen-new-domains|S186]] (91513c92) · Jebrim**

When the principal bundles a **direct question** with a task ("do X — but tell me, which Y…?"), answer the question as a **clean, standalone, foregrounded beat**, not folded into the work announcement. If the answer is buried under task narration, he has to re-ask.

**Anchor.** This session the principal asked *"which things looked like they deserve a domain of its own that we don't already have?"* I gave the answer — but inside a long message that led with the nfe-repo digest re-synthesis, so the ranked verdict was buried below the fold. He came back: *"ok so we have the nfe repo, but what about the question i asked - what other things deserve a topic of their own?"* The answer existed; it just wasn't surfaced as its own thing. I then re-delivered it cleanly (test stated, ranked, with fold-vs-standalone calls) and it landed.

**Why it matters.** A question carries its own intent separate from the task. Burying the reply costs a full round-trip and reads as not-listening. The Understanding/Plan preamble confirms the *ask*; it doesn't substitute for *answering the explicit question first, visibly*.

**How to apply.** When a turn contains both a task and a question: lead the response with the question's answer (or break it out under its own heading), *then* the task work. The compression rule still holds — short is fine — but the answer must be the foreground, not a clause inside a build announcement.

Sibling of [[feedback_act_only_when_asked_to]] (a question is a request for an answer, not a license to start building) and the content-over-verbosity / lead-with-the-surface lessons.
