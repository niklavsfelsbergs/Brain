# S279 — 2026-06-26 — cockpit click-to-open: resolve agent-relative paths

- Niklavs: clicking a file-path link in the cockpit terminal flashed *"couldn't open ups_retention_levers.md — not found"* though the file exists. Diagnosed evidence-first (read-only): the [[S270_cockpit_click_to_open_link_reliability|S270]] `#pathlink-flash` toast firing was the tell — the link **matched** and the click **reached** [[backend.py|cockpit/backend.py]] `api_open_path`, which returned a real 404. So this was not the matcher ([[S270_07bbe4c6_cockpit_click_to_open_link_reliability|S270]]'s domain) but the **resolution** tier underneath it.
- **Root cause:** the printed path was **agent-relative** — `2_analysis/ups_retention_levers.md`, relative to the claude Bash cwd deep inside `bi-analytics-main/NFE/projects/2_EU_tender_2026/`. `api_open_path` only tried `BRAIN_ROOT/<rel>` then `WORKSPACE_ROOT/<rel>`; neither roots at the agent cwd, and no per-session cwd is captured anywhere, so every relative path printed from a non-brain working dir dead-clicked (most of Jebrim's output, since he lives in `bi-analytics-main`).
- **Fix** ([[backend.py|cockpit/backend.py]], code-only): a third resolution tier. When both direct candidates miss a relative path, new `_workspace_suffix_search` walks `WORKSPACE_ROOT` for a file whose path ends with the **segment-aligned** relative path; one hit opens, several return a 409 `ambiguous (N matches)` flash rather than guess; heavy dirs (`.git`, `node_modules`, `.next`, `__pycache__`, `.venv`, build output) pruned, match collection capped at 4. Security envelope unchanged — every match is under `WORKSPACE_ROOT` by construction.
- **Verified:** `py_compile` clean; [[test_backend.py|cockpit/test_backend.py]] **84/84** (+5 new checks — one-match resolve, deep-file identity, segment-boundary, ambiguity, pruned-dir exclusion); full dev batch `run_tests.py` **21/21 suites**. **Live smoke test** against the real workspace with the exact screenshot path → **exactly one match**, the real `bi-analytics-main/.../2_analysis/ups_retention_levers.md`. The walk took **~2.3s** (not "sub-second" as first claimed — workspace tree is large even pruned), but it fires only on the otherwise-dead-click fallback path, so a 2s wait that ends in an open beats an instant dead-click.
