# A broad file-selector re-evaluates against live state — name explicit targets in a busy parallel tree

**Observation ([[S214_03c3dbd4_g1_alching|S214]] alch, sid8 03c3dbd4, 2026-06-11).** Running an alching pass over a shared git tree with ~13 parallel sibling sessions live, I twice let a *broad selector* capture work that wasn't mine — caught both, but both were avoidable:

1. **A date-wildcard re-globbed a sibling's mid-pass write.** My promotion loop used `examine/drafts/2026-06-11-*.md`. Between the approval snapshot (19 files) and the loop, the live sibling `12aa2f0f` wrote *and committed* its own `name-the-gate-...` draft into that same folder. The wildcard re-evaluated at execution time and swept the 20th file into `confirmed/`. Caught only because the count came back 20 ≠ 19.

2. **A directory-level commit pathspec snapshotted the working tree.** `git commit -- gielinor/players/jebrim/` does **not** commit "my staged changes under that dir" — it snapshots the *current working-tree state* of every tracked file under it, so it swept in unstaged deletions that weren't mine (a sibling's half-promoted 06-09 drafts, 32 sibling inventory-hygiene deletions) — 93 files instead of ~42. Caught by reading the commit's file list; reset (unpushed) and re-committed with explicit file-level pathspecs.

**The lesson.** A glob or a directory pathspec is a *query that re-runs against live state*, not a frozen list. In a single-actor tree that's harmless; in a shared tree with concurrent sessions it silently captures siblings' concurrent writes and unstaged deletions. **Snapshot the intended set explicitly and operate on named members** — enumerate the exact files at decision time and act on that list, and for commits prefer explicit per-file pathspecs (or verify the file-count against the expected number) over a directory pathspec. The cheap guard that saved both: **check the operated-count against the expected-count** before trusting a batch file-op.

This is two instances of one pattern in a single session, which is what makes it nameable rather than a one-off. Sibling-family of the confirmed [[2026-06-11-rotation-overwrite-clobbered-sibling-archive]] (whole-file write clobbering a sibling) and the pathspec-commit discipline already in MEMORY (`pathspec-commit-from-first-commit`) — this extends both to *read-side selection*: the danger isn't only what you write, it's what your selector picks up.
