[[cockpit/web/main.js]] keeps `<Term>` mounted (hidden) across the toggle so the PTY never drops; [[cockpit/backend.py]] gained `&full=1` to lift the 4000-char tool-result cap (a truncated copy is a broken copy). CSS + bigger transcript font in [[cockpit/web/styles.css]].

- **Lost-rename-on-reopen fix.** Renames are keyed by `sid8`, but `claude --resume` (cockpit reopen) mints a NEW id (unlike `--session-id`, which pins) → the label is orphaned. The old `_carry_name_across_clear` covered only in-process `/clear`, only disk, and baked in a false "resume reuses the UUID." Now carried across the rotation in **both** stores: localStorage in [[cockpit/web/term.js]]'s re-announce handler, disk in new `_carry_disk_name` in [[cockpit/ptybridge.py]].
