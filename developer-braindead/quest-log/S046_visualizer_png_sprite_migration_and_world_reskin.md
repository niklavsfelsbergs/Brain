# S046 — 2026-05-22 — Visualizer PNG sprite migration and world reskin

Long iterative session migrating the visualizer from procedural SVG art to PNG sprites generated by ChatGPT image-gen. Four sprite-sheet attempts landed before a coherent style stuck. Lots of throwaway intermediate work captured in `experiments/visualizer/sprites/_source/`.

- **Four sprite-sheet iterations.** v1 (`sheet-2026-05-22.png`, chunky 3D actors + buildings) had a JEBREM typo; v2 (`sheet-2026-05-22-v2.png`, slim full-body figures) replaced it; v3 (`tileset-2026-05-22.png`, 65 environment tiles) was layered on top but introduced a third art style that fought the v2 sprites — the visual collage made the principal call for a rework. v4 (`sheet-2026-05-22-v4.png`) committed to the chunky cartoon SVG-throwback aesthetic in one coherent 27-sprite sheet. v4 is the one that landed.
- **Slicer (`sprites/slice.py`) evolved through four passes** as artifacts surfaced. Hard-coded `ROW_Y_RANGES` per sheet (auto-detection didn't generalize across sheet shapes). Added `edge_alpha` with two halo-eating layers: aggressive `BG_TOL=60` flood-fill, plus `dilate_bg(3)` that unconditionally eats the brightest halo ring. The user-visible white edges around sprites were 100% slicer fault, not ChatGPT — diagnosed by sampling first-opaque pixels in `gnome.png` at three Y positions and seeing RGB ~(151,153,152) with full alpha; resolved by combining higher BG tolerance + mask dilation.
- **`developer-braindead/experiments/visualizer/index.html` heavily edited.** Five actor `<g id>` symbols (jebrim, zezima, wisp, braindead, guthix) swapped from inline SVG primitives to `<image>` refs. Three sub-agent spawn functions (`spawnDwarf`, `spawnGnome`, `spawnPenguin`) rewritten — per-spawn color tint dropped (PNG can't be re-tinted cheaply); ID badge becomes the only per-instance distinguisher. The 529-line procedural `buildBuildings()` collapsed to a 38-line PNG loop driven by `BLD_SPRITE_DIMS`. Procedural ground swapped first for tiled v3 ground blocks (visible seams between tiles — principal asked to drop), then for a single solid green rect (`#47771c`, sampled from `sprites/ground-tile.png`). `buildPaths()` call commented out per principal request — paths overlapped sprites and looked rough; flora's `clearOfPaths` checks now no-op trivially. Helper functions `isoBuilding`, `wallTexture`, `roofTexture`, etc. are now dead code (~700 lines) — sweep next bankstanding.
- **Bubble + label positioning recalibrated for taller v4 actors.** Closed bug #2 deferred from [[S042]] (`SPRITE_HEAD_Y=-8` calibrated for unscaled sprites; bubble pointers anchored at empty air): bumped `GATHER_SLOTS.bubbleY` by -32 across the board so bubbles sit above the head with v4 sprites at scale(2.4). Also recomputed `LABEL_Y_OFFSET` per building using `yoff = 41 - sprite_height` so name plates land just above each building's PNG top edge instead of floating at the old procedural roof+spire height. Bumped flora clearance pads (`bPad` 84→140, `pathPad` 25→40, tree clearance 120→160) since v4 buildings are larger.
- **Lessons worth flagging.** (a) Mixing PNG sheets from different ChatGPT generations is a coherence trap; even with style instructions, multiple sheets drift. Commit to one sheet per art direction. (b) The "regenerate if detailed" instruction in the prompt is the load-bearing line for getting the chunky SVG-throwback aesthetic — ChatGPT's default is much richer. (c) PNG halos at sprite edges read as "the AI generated bad sprites" to the principal; framing the diagnosis ("100% my slicer's fault, not ChatGPT") mattered more than the fix. (d) `image-rendering: pixelated` on `svg.map image` doesn't fully inherit to dynamically-spawned `<image>` elements in all browsers — apply via attribute AND inline `style=` on spawn paths.
