Multiple-choice → built the neuron-overlay, then iterated hard on principal feedback across the turn. **RUNTIME-UNVERIFIED** (no GUI eyeball from here; verified every testable layer headless).

- **Built `cockpit/graph-export.py`** — reuses the exact Obsidian resolver from [[obsidian-graph-report.py]] to export the whole vault as `cockpit/web/graph.json`: **690 nodes / 1742 resolved `[[wikilink]]` edges**, each node tagged with a `group` (color) and a `cluster` (cognitive area = player × layer). Regenerate with `py cockpit/graph-export.py`. The graph.json is a static artifact (goes stale as the brain grows).
- **Built `cockpit/web/brain.js`** — self-contained ES module, mounted into the feed column by `feed.js` (one `<div class="brain-dock">` + a once-`useEffect` `mountBrain` with teardown). Sizes its canvas off its real rendered box each frame → **agnostic to the `.app-grid` zoom:1.35** (sidesteps the [[S069_c3f2e3f3_terminal-scale-fix-and-brain-presentation|S069]] landmine instead of fighting it). No new hook, no backend change, no deps/build — reads the already-polled `/api/feed`.
- **Iterated through four shapes in one turn** (principal feedback each step): (1) abstract region cartoon — *"bruh this is useless"*; (2) real force-directed graph of all 688 nodes; (3) docked as a square at the feed top; (4) **3D** with hand-rolled rotation/projection (no Three.js) + left-pan/right-rotate + 6 FX (neon bloom, signal pulses, comet trails, depth fog+vignette, organic edges, region hulls) — *"too colorful and polygonal"*; → (5) the **two-level bubble map** (principal pick).
- **Final design — two levels.** OVERVIEW: ~25 labelled **area bubbles** (`jebrim·quest-log`, `guthix`, `identity`, `dev·bank`…; singletons fold to `·misc`), **calm parchment at rest**, a bubble glows **in color only when a note inside it fires** → reads *where* the brain is working. Click a bubble → EXPAND that area's notes as a force graph (bloom + traveling synapse pulses + comet trails); `← back` returns. Fixes both complaints by construction: **color = activity** (not decoration), **no hull polygons**.
- **Verified headless** at every step (`node --check` + DOM-stub harnesses loading the real `graph.json`): final bubble-map harness **22/22** — clusters assigned + singletons folded, ~25 non-overlapping bubbles, inter-cluster connectors, drill-in builds correct local sub-graph, firing lights node + bubble + spawns pulses, back returns, teardown clean. Earlier shapes: 13/13 (2D graph), 12/12 (3D sim bounded + rotation orthonormal), 8/8 (dock mount). What stays unverified is purely visual: label legibility in small bubbles, calm/readability, perf at 690 nodes.
