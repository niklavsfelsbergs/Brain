# Niklavs scopes analytical methodology to the analyst character

**Observation, S014 T23 (2026-05-21).** I surfaced the drill-to-disprove meta-rule (publish-aggregates-only-after-disprove-pass) and proposed routing options — Jebrim's `examine/` versus global `examine/`. I argued the rule generalized beyond mart work ("would catch the same class of error on revenue mix, error rates, cost attribution") and tentatively recommended Jebrim's `examine/` first, lift to global later.

Niklavs replied: **"Don't make this global - Jebrim is the analyst."**

The signal is twofold:

1. **Analytical methodology lives with the analyst character, not the system as a whole.** Even when the rule has plausible cross-domain applicability (Zezima could hit the same shape on personal-life data), Niklavs doesn't pre-route it global. The rule's home is wherever its practice lives — Jebrim does analytical work, so Jebrim owns the discipline.
2. **He resists premature generalization in identity routing.** I framed it as "lift to global later if Zezima hits the same shape" — implying the global slot is reserved for cross-player patterns proven across players. Niklavs's edit was crisper: skip the "lift later" caveat entirely. The promotion path is bankstanding's job, not the per-session router's.

## How to apply going forward

When I'm tempted to route a finding global because the *rule* sounds general, ask first: *whose practice does this live in?* If it's bound to a character's domain (analyst, personal-life, builder), keep it character-scoped. Let bankstanding find cross-cutting patterns later — that's literally what bankstanding's read-across-all-players capability exists for.

This also means: when in doubt, route per-player. The cost of misrouting global is structural (the identity layer carries content that belongs in a character), whereas the cost of misrouting per-player is just a future bankstanding lift if a real cross-cutting pattern emerges.
