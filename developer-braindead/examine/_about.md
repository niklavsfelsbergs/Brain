# `examine/` — about

Self-model for dev-Claude. Where I accumulate how to act and how to develop in collaboration with Niklavs on this project.

Metaphor: the "examine" verb — the agent inspecting itself. Different from `player/` (which models the principal).

## What goes here

- **Identity entries** — `I-NNN_descriptive_name.md` per entry. Postures, rulings, and patterns about how I work *here*, with this user, on this project.

## What's elided here vs. main brain's `examine/`

The main brain's `examine/` is the running agent inspecting itself in operation — capabilities, current loaded personality, ongoing self-observations. The dev brain's `examine/` is narrower: just *how dev-Claude works with Niklavs during build sessions*. No runtime self-state — there's no runtime.

## Relationship to main brain personality

Some identity entries may eventually export to the main brain's personality file as the agent's posture inherits the patterns I've learned working with Niklavs. Others stay dev-only (e.g., procedural patterns specific to dev sessions). The export step is a deliberate move per [[D-001]], not automatic.

The original `PERSONALITY_LEDGER.md` framing was "feeder to main brain"; that framing generalized in [[D-006]] to "dev-Claude's accumulated identity, some of which may export."

## Conventions

- One file per [[I-NNN]] entry. Stable IDs, descriptive filename suffix.
- Format per `spellbook/entry-formats.md`.
- Capture *as they happen* — corrections and validations both. See [[I-001]] (which started life as a single datapoint and is now the seed of the layer).
- Don't try to spec identity fully upfront. Iterative, like teaching a person.
