# A clean synthetic validation can hide population + small-n bugs that only live data exposes

**Observation.** This session I wrote a synthetic old-vs-new probe for the transit-tier serving re-point (`_verify_routes_sql.py`) — generated multi-status, fat-tailed, multi-corridor data and asserted counts/avg/percentiles/unlogged matched. It passed clean. Then the live old-vs-new on real 2025-12/2026-05 **failed two ways the synthetic pass was structurally blind to:**

1. **Population shape.** ~43% of real rows have NULL `current_shipping_status` (untracked shipments). My synthetic generator gave every row a status. The old SQL's `status <> 'DELIVERED'` drops NULLs via three-valued logic; the tier's `shipments-delivered-exception` re-included them → completeness/kpis unlogged off by ~7×. Fix: added an `inflight_n` tier column counting only known-non-terminal status.
2. **Small samples.** My synthetic corridors all had healthy n; real data had 51/137 corridors with <10 deliveries. Binned-CDF percentile reconstruction can't match `quantile_cont`'s between-points interpolation on sparse data → heatmap per-corridor p95 off up to 20 days. Fix: heatmap stays on per-shipment processed.

**Why it matters.** The synthetic probe tested the *mechanism* (does the formula compute correctly) and proved it to 1e-13 — genuinely useful, caught nothing wrong because nothing mechanical was wrong. But correctness over a *realistic distribution* is a different question, and my hand-rolled synthetic data encoded my *assumptions* about the distribution (clean statuses, decent n) — so it could only ever confirm those assumptions. The bugs lived in exactly the places my assumptions were wrong.

**Rule.** Synthetic/mechanism validation proves the math; it does not prove the data-shape assumptions. For any re-point/refactor over real data, treat a clean synthetic pass as necessary-not-sufficient and run the live old-vs-new before declaring it correct — and bias the synthetic generator toward the *adversarial* shapes (nulls, empties, tiny groups, fat tails), not the convenient ones.
