# A code docstring describing the data is not ground truth — verify against the data

**Observation (S150 e04c495d, 2026-06-05).** Building the "vs today" baseline, I needed to know which carrier carries Maersk's FR volume. The `maersk_eu.py` engine **docstring header** said: *"Maersk is the contract intermediary; last-mile is run by national partners (FR: Colis Prive; …)."* I took that at face value and told the principal FR-Maersk = Colis Privé (the 107k `CPRHD` group). The principal corrected me: **"FR Maersk will have `shippingprovider_extkey` MAERSKFR. The last mile carrier Maersk deals with themselves."** Checking the actual mart coding confirmed him — `MAERSKFR` is its own extkey (107 parcels in 2025, 27,624 in 2026-Q1); Colis Privé (`CPRHD`) is a *separate* direct contract. The engine header was wrong (or described a superseded arrangement).

**The trap.** A docstring / code comment *describing how the data is structured* reads as authoritative because it's in the codebase — but it is a **secondary description**, not the data. It drifts, predates changes, or was wrong from the start. The authoritative source for "which carrier is this shipment" is the mart's own field (`shippingprovider_extkey`), not a sentence in an engine that happens to mention carriers.

**Why this is load-bearing.** Same family as *read-domain-knowledge-before-proposing* (S124) and *verify-the-thing-don't-trust-the-wiring* (keepsake), but a sharper edge: the misleading source was *inside the code I was already reading*, which makes it feel verified when it isn't. The cost here was bounded (the principal caught it in discussion), but the wrong carrier-identity would have mis-attributed the entire FR "vs today" number.

**How to apply.** For any "which entity / which carrier / which category does this row belong to" question, resolve it against the **authoritative data field** (the mart's extkey/group, per the shipping-agent contract), not a code docstring, variable name, or comment that describes the mapping. Treat a code-comment's description of the data as a *hypothesis to check against the data*, exactly like a memory claim. When the comment and the data disagree, the data wins (and the comment is a defect to flag).
