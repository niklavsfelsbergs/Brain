# S134 — 2026-05-31 — Cockpit board UX: last-action sort, busy keeps highlight, ritual status chips

Dev-brain via "lets develop gielinor", mid-conversation. Principal wanted a collaborative UX audit of the cockpit/switchboard board — two known pains plus a sweep for what else was rough, including which rituals lacked a status surface. Grounded read-only first ([[board.js]], [[main.js]], [[feed.js]], [[backend.py]], [[status-sidecar.py]]), surfaced a ranked diagnosis, then built four fixes on the principal's multiple-choice pick (all four).

- **#1 same-status sort + IDLE "0s" (two bugs).** Root cause: `backend.py build_session_model` sorted `(rank, age_sec)` — `age_sec` is session *lifetime*, so same-status rows ordered oldest-launched first, not by last action; and cockpit-own (unmanifested) rows in `main.js` hardcoded `age_sec: 0` and were appended *after* the backend's sorted list with no re-sort. **Fixed:** backend now emits `last_action_ts` (the heartbeat) and sorts `(rank, −last_action_ts, age_sec)`; `main.js` gives cockpit-own rows a real `last_action_ts`/`quiet_sec` from `fs.ts` and **re-sorts the merged list**; `board.js` age chip now shows time-since-last-action (lifetime → hover title) so the displayed number is the sort key.
- **#2 busy never loses highlight + #4 wired the dead `stalled` state.** The blanket `quiet > 5min → grey` (`stale`) applied to *every* state incl. `busy` — the [[S083_d71c4ab3_cockpit-full-audit|S083]] long-turn false-trip in display form. **Fixed:** greying now reserved for genuinely not-live rows (`idle`/`done`/`ended`); `your_move` quiet past `IDLE_AFTER_SEC` → `idle`, `busy` quiet past a **generous** `STALL_AFTER_SEC=900` (15 min) → `stalled` (the previously-unassigned chip — crash/reopened-cockpit signal, surfaced + pulsing, never greyed). `needs_you` never decays.
- **#3 ritual status chips.** `status-sidecar.py` `MODE_VALUES` extended with `bankstanding`/`consultation`/`drafts` (flavor `FLAVOR_MODES`, tag on busy like alching); `board.js` `TAG_LABEL` + `styles.css` flavor pills (Guthix rituals gold, drafts purple). The marker-*writing* half is user-only ritual prose — applied on the principal's explicit "a": [[bankstanding]] (entry/Phase-0-restore/close), [[drafts-triage]] (entry/exit), [[guthix]] Visualizer (consultation/bankstanding mode marker), [[communication-protocol]] Mode-marker section (3 new tokens + corrected the stale precedence-ladder note to the [[D-029_switchbar_two_axis_states|D-029]] flavor-tag model).
