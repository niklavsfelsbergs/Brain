# S188 — 2026-06-11 — Switchboard state-derivation fixes (busy / idle / stalled)

- **Diagnosed three cockpit session-state bugs Niklavs reported** (all cockpit-driven rows), instrument-first against live state rather than re-theorizing: (1) BUSY when it's his move, (2) a finished turn flips to IDLE almost instantly, (3) a session shows STALLED while actively working. Caught (1)+(3) live on the **same UPS session** (`01871b26`, intent *"ups-2.0.0 SHIPPED…"*) — `state=busy tags=['monitoring'] last_event_kind=Stop`, and a manifest `latest_action_ts` 23 min stale while `state.ndjson` had an `editing` action 1s old. Root across all three: the cockpit has **no "actively generating" signal** — it infers liveness from tool *completions* (`emit-event.py` stamps the action heartbeat on **PostToolUse** only) + Stop, so a long single op/generation freezes the heartbeat and a launched bg-task is mistaken for a busy wait.
- **Bug 1 — fixed + live-confirmed.** Retired the [[S141_98194a37_cockpit-board-auto-title-wrap-states-bg-wait|S141]] `your_move → busy` flip in [[status-sidecar]] (`status-sidecar.py`): launching a `run_in_background` command and ending the turn is *the principal's move*, not a busy wait — the flip false-pinned BUSY on every turn that spawned a monitor/build. `monitoring` now rides as a **sub-bubble** under the real ball-state. Wired through [[backend|cockpit/backend.py]] (monitoring → sub), `board.js` + `styles.css` (label + crew-coloured style). `01871b26` re-read **your_move** on its next Stop under the new hook — fix is live at the hook tier immediately (hooks run as fresh subprocesses).
- **Bugs 2 + 3 — fixed (frontend, runtime-unverified until cockpit relaunch).** Gave cockpit-driven rows a **ground-truth liveness signal**: the cockpit owns the PTY byte stream, so [[term]] (`term.js`) stamps `_lastOutputAt` on every server→terminal write and exports `termLastOutput(sid8)`; `main.js` `ptyLiveness()` recomputes the STALLED/IDLE liveness from output recency instead of the frozen backend heartbeat. A streaming turn never reads STALLED (3); the IDLE clock counts from the **last output = the response ending**, not a mid-turn event (2). No-op for VS-Code / observed rows (falls back to the backend).
- **Regression gate green.** `test_backend.py` **74/74** — new `test_monitoring_is_subbubble` (your_move + monitoring → main YOUR MOVE, monitoring sub, attention true) + relabelled the old busy+monitoring case as a legacy-shape guard (a pre-S188 status file must still never stall). `node --check` on the 3 JS files + `py_compile` clean.
