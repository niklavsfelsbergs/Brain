# S033 — 2026-05-22 — Visualizer audit (live mode + hooks)

- **Scoped audit** of live-mode visualizer + hook scripts (`emit-event.py`, `status-sidecar.py`, `emit-commit-event.py`, `experiments/visualizer/index.html` live-mode + switchboard paths). Replay mode and CSS/visual chrome out of scope this pass. Produced a 14-item triage list grouped HIGH (2) / MEDIUM (5) / LOW (7); entry below tracks landings as they happen.
- **HIGH findings.** (1) Sidebar structurally broken right now — sidecar registers only `SessionEnd`, so a working session never writes a status file until it ends and `deriveState` (index.html:4315) flips everyone to `idle` after 5min. The pulsing-yellow waiting-for-user signal — the whole point of Phase 2 — fires for no one. (2) Sidecar reports `instance: 1` for every session (status-sidecar.py:246 uses `prev.get("instance") or 1`, never reads `state-instances.json`). Two parallel Braindead sessions render identically as "Braindead" in the sidebar.
- **MEDIUM findings.** (3) `.tmp.<pid>` files leak when `os.replace` fails on Windows — two scripts writing the same dir + concurrent sessions. Evidence on disk. (4) `state.ndjson` grows unboundedly; `pollLive` reads whole file at 500ms. Future perf cliff — **deferred per session ruling**. (5) `state-actors.json.byId` has no GC on SessionEnd; 8 dead Braindead sessions + 6 dead Jebrim sessions accumulated. (6) Sidecar's actor candidate list is hand-synced with `emit-event.py`'s roster. Adding a player means editing two files. (7) Stale-Braindead `_mode_session_id` not cleared on SessionEnd; recovery silent.
- **LOW findings.** (8) `braindead-ed610cbe` legacy event leak — `parseInstanceKey` regex needs digit-only suffix; sid8 hex suffixes fall through and fire "no spawn path" warn. (9) `_write_manifest` docstring lies — says "Excludes 'ended' entries" but doesn't. (10) Intent staleness carried forward when actor detection returns "unknown". (11) `despawnPlayerInstance` fades sprite over 500ms but snaps bubble instantly. (12) `gc_stale_subagents` per-call cost — **deferred**. (13) `bash_subtask` always mislabels `echo` as "writing sidecar". (14) Dead code from [[S031_visualizer_world_scale_layout_and_bubble_redesign|S031]] (`updateDayNight` et al, `#day-night` rect already deleted).
- **Session ruling on three open items.** Registration shape (#1) → lower-frequency (UserPromptSubmit + Stop + SessionEnd). Manifest filter (#9) → land the filter, hide `ended` sessions. Perf hazards (#4 + #12) → defer; not biting yet at 2378 ndjson lines.

## Landings

- **Pass A — index.html (#14, #11, #8).** Deleted dead day-night code (`DAY_NIGHT_ANCHORS`, `currentHour`, `dayNightFill`, `updateDayNight` + the two call sites in `seekTo`/`tick` + the leftover CSS comment block referencing the removed `#day-night` rect). `despawnPlayerInstance` now passes `false` to `clearIntent` so the bubble fades over 360ms alongside the sprite's 500ms fade-out rather than snapping off. `ensureActorExists` gained a sid8-suffix strip for legacy `<actor>-<8hex>.txt` actor names that `parseInstanceKey`'s digit-only regex falls through; the `braindead-ed610cbe` historical event from state.ndjson:5 now resolves to bare `braindead` and stops warning.
- **Pass A — emit-event.py (#13, #5, #7).** `BASH_SUBTASK_TABLE`'s `echo` pattern now requires `> .claude/<sidecar>` to claim "writing sidecar"; plain `echo "hello"` falls through to the generic "running shell command". `handle_session_end` extended to clear this session's keys from every `state-actors.json` `byId` map (finding #5 — 8 dead Braindead + 6 dead Jebrim entries observed pre-fix). Also clears `_mode_session_id` when this session owns the marker (finding #7), so a crashed dev-brain session leaves the next dev-brain write room to re-take ownership.
- **Pass A — status-sidecar.py (#6, #10, #3).** `_detect_actor` replaced hardcoded `("jebrim","zezima","braindead","guthix","wisp")` candidate list with a glob over `intent_dir/*-<sid8>.txt`; new players land without a sync edit. Intent carry-forward only inherits `prev.intent` when `prev.actor == current_actor` — an "unknown" early-fire no longer pins a prior actor's intent in the sidebar. New `_sweep_stale_tmp()` removes `<base>.<ext>.tmp.<digits>` files older than 5 min from `VIZ_DIR` + `STATUS_DIR`; runs after `_sweep_stale` on every fire. Confirmed clean: pre-fix had 5 leaked tmps; post-fix run shows zero.
- **Pass B (#1, #2, #9).** Settings.json already carried the lower-frequency registrations (`UserPromptSubmit + Stop + SessionEnd`) when this session opened — `_comment_status_sidecar` cites the S033 decision; treat as pre-landed. Sidecar now calls `_detect_instance(actor, sid)` against `state-instances.json` so parallel sessions of the same actor render as e.g. Braindead·2 in the sidebar instead of two indistinguishable Braindead rows. `_write_manifest` excludes `state == "ended"` entries — code now matches the docstring promise.
- **Deferred per ruling.** #4 (state.ndjson unbounded growth) + #12 (`gc_stale_subagents` per-call cost) — not biting at 2378 ndjson lines; park as future tickets.
