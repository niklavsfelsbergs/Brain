# R-001 — Dev↔main export discipline failure → main brain pollution

**Severity.** high. Opened in [[S001]].

The whole point of two-brain separation ([[D-001]]) is to keep dev narrative out of the main brain. If the export step is sloppy — copying session notes wholesale, importing dev rationale into main artifacts — the separation collapses.

**Mitigation.** Every quest-log entry ends with **Main-brain changes.** stating exactly what crossed and why. Each main-brain edit must be defensible as *current state*, not *history*.
