# S049 — 2026-05-23 — Visualizer state-aware motion and switchboard action line

[[S048_b070e9be_visualizer_manifest_driven_sprite_sync|S048]] inverted the visualizer to manifest-driven but the principal observed sprites still standing still and switchboard rows showing intent from earlier in the session that never refreshed. Three parallel dwarves diagnosed and fixed the three failure modes; a follow-up inline edit added state-awareness to the wander so working sprites visibly behave differently from idle ones.

- **Intent doc drafted before any spawn.** The map is a renderer of three signals: **intent** (agent-narrated, low cadence) on sprite bubble + sb-row line 1; **action** (hook-emitted per tool call, high cadence) on sb-row line 2; **liveness** (state-class glow + ambient wander, per 2s poll) on the sprite itself. Pre-spawn invariants captured in `quest-log/in-progress/S049_17e701eb_map-fixes.md`.
- **D1 — wander animation root cause.** `.wander { transition: transform 1200ms }` at `experiments/visualizer/index.html:305` targets the CSS `transform` property; the JS at three sites was writing the SVG `transform` attribute (`setAttribute('transform', ...)`), which CSS transitions don't see on this code path. Result: instant teleports every 4–12s, looked frozen between hops. Swapped all three sites to `style.transform = \`translate(${dx}px,${dy}px)\`` (`wrapWander` init, `tickWanderForActor`, `moveActor` clear). Sibling: `quest-log/in-progress/S049_17e701eb_d1_wander.md`. Flagged but out-of-scope: `setActorTransform` shares the same anti-pattern — works in current browsers but lives on borrowed time.
- **D2 — switchboard action line.** Added `latest_action` + `latest_action_ts` to per-session manifest records via `_read_state_tail` + `_format_action` + `_latest_action_index` in `.claude/hooks/status-sidecar.py`. Renderer in `index.html` got a 3rd grid row `.sb-action` showing `Tool · target · Ns ago`, age refreshing per second alongside `.sb-age`, hidden when absent. Decouples row freshness from intent narration cadence — agent doesn't need to narrate more often; the action line ticks at hook-fire frequency. Sibling: `quest-log/in-progress/S049_17e701eb_d2_switchboard-action.md`.
- **D3 — intent-refresh verification.** Read-only smoke test: imported `status-sidecar.py` and called `_detect_actor` post-intent-file-edit; returned fresh content. Manifest re-reads intent files fresh on every sidecar fire (no cache, no skip). Concrete fire-boundary observation: this parent's intent flip from `"Clarifying map intent + dwarf briefs"` → `"Spawning S049 map-fix dwarves"` propagated within one fire (file mtime 1779490697, manifest write 1779490807). **Verdict: plumbing is clean; the "freeze" symptom was purely hook-fire cadence (3 fires/turn since [[S037_terminal_switchboard_phase_3_click_to_focus|S037]]) — exactly what D2's action line solves.** Sibling: `quest-log/in-progress/S049_17e701eb_d3_intent-refresh.md`.
- **State-aware wander (inline follow-up, no dwarf).** Post-dwarves, principal observed sprites still felt inert when working — flat 4–12s cadence with ±18px range left them still 80%+ of the time. Replaced flat `WANDER_MAX_PX` / `WANDER_MIN/MAX_INTERVAL_MS` constants with `WANDER_STATE_TABLE` keyed by sprite state-class: **working ±35px every 1.2–3.5s** (energetic — drifting nearly continuously), **waiting ±18px every 4–12s** (calm, unchanged), **closing ±10px every 6–14s** (winding down), **idle/ended frozen**. Added `wanderParamsFor(g)` helper reading the sprite's current `state-*` class; modified `tickWanderForActor` + `maybeTickWander` + `moveActor` to use per-state params. `setSpriteState` now clears the wander timer on state-class change so transitions feel responsive within ≤1s instead of finishing out the previous schedule.
