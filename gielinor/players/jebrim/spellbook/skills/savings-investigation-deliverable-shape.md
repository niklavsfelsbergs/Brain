# Savings / optimization investigation — deliverable shape

> **Status.** Draft. Authored 2026-05-23 (Jebrim), harvested from a shipping-agent quota-reduction conversation.

## When to use

Any investigation whose ask is *"find a way to achieve X"* / *"where can we save"* / *"how do we cut Y"* — the deliverable is a set of **moves**, usually a ranked lever menu.

## Two rules

### 1. Lead with moves, not sizing

The ask "find a way to drop the quota" wants *what to do, sequenced* — not a money-sizing menu. Sizing answers "how big"; the question asked "what do I do." Lead with the ranked actions (each: what to do, why, rough effort), and attach the sizing to each action as support.

**Anchor:** in the originating session I delivered a 6-lever €-sizing menu first; the principal had to ask *"where are the actual suggestions."* The sizing was correct but it wasn't the deliverable — moves were.

### 2. Net out overlap before headlining a stacked total

When levers are summed, they double-count — the same parcels recur across multiple levers. Don't headline the additive sum as achievable. Lead with the de-duplicated realistic figure; present the additive total only as an explicitly-labelled upper bound.

**Anchor:** same session — stacked 6 levers to 4.05 pp additive, then walked it back to "3.0–3.5 pp realistic, subtract 20–25% for overlap." The realistic number should have been the headline; the additive sum is misleading on first read.
