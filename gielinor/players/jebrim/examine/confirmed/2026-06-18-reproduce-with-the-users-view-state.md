# Reproduce a UI-over-data bug with the user's actual view state, not the default

**Observation ([[S264_17290ea4_scm-shift-tabs-baseline-prune|S264]], 2026-06-18).** On the SCM "cost drivers shift tabs show `--`" report, I traced the full stack — code, local parquet, live parquet, deployed-vs-local code, default date window — and every path I could reproduce rendered cost correctly. I was about to conclude "the data/query/code are all fine; this must be a live-runtime or browser-cache issue I can't see" and route the diagnosis to the serving pod / stale bundle. The bug was reproducible the whole time — I just wasn't reproducing it under the **filter state the user actually had**. The principal's screenshot URL carried `order_srcs=PCS,PicaAPI,Picturator`, and that order-source filter is exactly what flips the query onto a different data tier (`processedAsDaily`) where the real bug lived. With the filter applied, it reproduced instantly.

**The failure mode.** For a UI-over-data bug, "I can't reproduce it from the default view, so the data/code must be fine" is a false conclusion when the bug is **conditional on a filter, sort, date range, or toggle that routes to a different code or data path**. The default view exercises one path; the user's view may exercise another. Concluding "fine" from the default view asserts an absence I never tested.

**How to apply.** Before concluding a UI-over-data bug isn't reproducible (and especially before blaming runtime/cache/environment), get the user's **exact view state** — the URL with its query params, the active filters, the selected period/sort/toggle — and reproduce under *that*. Ask for the URL or a screenshot early; it's the cheapest disambiguator. In a tiered/branchy backend, enumerate which filters change the code path (here: site/shop/order-source → processed tier) and test each, rather than testing only the default. This is the never-assert-absence reflex applied to repro state, and a sibling of distinguish-fixture-from-live: the default view is itself a kind of fixture.
