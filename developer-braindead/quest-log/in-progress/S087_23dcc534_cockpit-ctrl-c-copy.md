# S087 — 2026-05-25 — Cockpit terminal Ctrl+C copy (+ feed-filter diagnosis)

Entered mid-conversation via "lets develop gielinor". The principal raised two cockpit reports while a live Jebrim session (`jebrim-249eb38a`, shipping-agent) ran in the PTY terminal beside the feed.

- **Report 1 — "messages not popping up in my feed" was a filter, not a bug.** Pulled the live `/api/feed` off the running backend (port 8770): the Jebrim session's events were all present (11 items in the payload — its `say` prose + `action` tool calls). A *working* session emits almost entirely `action` (130/300) + `say` (83/300) kinds, and both are gated behind the FEED panel's `actions`/`prose` checkboxes (`web/feed.js`). With those toggled off, a session shows only the sparse lifecycle checkpoints (picked_up/done/comms) — looks dead between PICKED UP and DONE. Both default **on**; they'd been switched off (localStorage-persisted). Diagnosis settled in one step by inspecting the actual payload rather than guessing at a render bug. **No code change** — operational (re-tick the boxes).
- **Report 2 — Ctrl+C couldn't copy from the terminal.** Root cause: [[term.js]]'s `attachCustomKeyEventHandler` handled Esc / Ctrl+V / Shift+Insert / Shift+Enter but **never Ctrl+C**, so xterm always sent raw `\x03` (SIGINT). No selection-aware copy path existed.
- **Fix (principal chose copy-on-selection, SIGINT fallback):** Ctrl/Cmd+C now copies when `term.hasSelection()` and swallows the keystroke; with no selection it falls through so SIGINT still interrupts (Esc already interrupts here too, so claiming Ctrl+C for copy costs nothing). New `_copyToClipboard` writes via a new backend `POST /api/clipboard` → `_write_clipboard_text` (ctypes, mirrors the existing reader — WebView2 gates the native `navigator.clipboard` write the same way it gates reads, which is why paste already needed the bridge), navigator fallback for the plain-browser case.
- **Verification:** `node --check` clean (term.js, board.js), `py_compile` clean, regression suite **39/39** (added a no-clobber `POST /api/clipboard` route assertion — empty text hits the guard before touching the real clipboard). Build tag `b86.3 → b87.1`. Principal relaunched and confirmed: **"works."**
