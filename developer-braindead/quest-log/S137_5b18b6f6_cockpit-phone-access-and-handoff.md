Principal's ask: *"how can I talk to my cockpit agents through my phone?"* — answered, then built the reachability, the mobile UI, and the takeover feature, iterating live against a real iPhone.

- **Remote access (committed `8f677ab`).** The cockpit is already a web app (aiohttp + Preact; `app.py`/pywebview just wraps `http://127.0.0.1:8770`), so the only barrier was the loopback bind. Made the bind **config-driven** in [[app.py]] — `config.json {"host":...}`, default `127.0.0.1` so a normal launch is byte-for-byte unchanged; opt into `0.0.0.0`/a tailnet IP for remote. Tailscale chosen as the transport (device-trust replaces the localhost boundary). `config.json` gitignored — a committed `0.0.0.0` would ship a wide bind as the default. Principal installed Tailscale on PC (`100.107.34.85`) + iPhone (`100.84.151.128`); verified HTTP 200 on the tailnet, phone reached it.
- **Mobile layout (committed `8f677ab`).** New self-contained `cockpit/web/mobile.css` + `mobile.js`, linked from [[index.html]] — does NOT touch the live-sibling `styles.css`/`main.js`. Below 760px the 3 columns stack into a Board/Session/Feed bottom tab bar; injected outside the Preact tree, body-class driven. Two live-caught bugs fixed: (1) **"doesn't fit"** = the default `--zoom: 1.35` scaled the session view past the viewport (`zoom: var(--zoom)` on console-head/composer) → neutralized `--zoom: 1` on the mobile body; (2) **"can't write" / "only transcript"** = the session opened was a **read-only VS Code peek** (`console.js` banner) — explained the cockpit drives only what it launched; also un-dimmed the reading-mode composer, scroll-to-bottom on open, auto-jump to the Session tab on select.
- **Cross-client handoff (this commit, S137).** "Take over a session driven on another client." Design forced by the PTY model (1 WS ⇄ 1 PTY ⇄ 1 claude; [[ptybridge.py]]): **kill-then-resume**, never two writers on one transcript. Five pieces: [[ptybridge.py]] `_LIVE_PTYS` registry by sid8 + `terminate_session()`; [[backend.py]] `POST /api/handoff` (token-gated like `/pty`, dev-guard-blocked); `cockpit/web/term.js` `handoffAndResume()` (the `found:false` gate refuses to resume if no cockpit PTY was killed); `cockpit/web/console.js` a "drive here" button on the peek banner, gated `host==="cockpit"`; `cockpit/web/main.js` carries host onto the peek + `doAdopt`. VS Code sessions can't be adopted (cockpit can't kill their claude) — button hidden.
- **Verified vs unverified.** Backend layer verified empirically on throwaway ports (live `:8770` untouched): no-token→403, valid-token+unknown-sid8→`{ok:true,found:false}` (the safety path), dev-backend `POST /api/handoff`→403 while reads stay 200. py_compile ×2 + node --check ×3 clean. **Cross-client runtime UNVERIFIED** — the full kill→resume→reattach loop needs two devices + a real session + a cockpit relaunch (the live backend holds pre-handoff code); mobile layout is live now (web/ served `no-store`, phone refresh).
