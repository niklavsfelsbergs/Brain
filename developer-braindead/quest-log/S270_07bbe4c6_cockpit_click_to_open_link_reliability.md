# S270 ‚Äî 2026-06-18 ‚Äî Cockpit click-to-open link reliability

- **Problem (Niklavs).** Clicking a file-path link in the cockpit terminal *sometimes* opened the file (HTML‚Üíbrowser, SQL‚ÜíVSCode) and sometimes did nothing ‚Äî felt random. He asked to "just make it reliable."
- **Diagnosis (read-only trace of the stack).** Click-to-open lives in `cockpit/web/links.js` (matcher + xterm `registerLinkProvider` + transcript `linkifyPaths` + a global delegate) ‚Üí `/api/open-path` in `cockpit/backend.py` (`os.startfile` / explorer-reveal, workspace-scoped). Three independent gaps produced the "sometimes": **(1) line-wrapping** ‚Äî the terminal provider scanned ONE visual row via `getLine().translateToString`, so a long path wrapped across two grid rows was only ever seen as a fragment ‚Üí no link (short paths fit one row ‚Üí worked; this is the dominant cause). **(2) bare filenames** ‚Äî the matcher required ‚â•1 `seg/`, so a path printed without a folder (`rollups.sql`) never linkified. **(3) silent backend rejection** ‚Äî a 403/404 from `/api/open-path` failed with no feedback, making a dead click indistinguishable from "no link there." The default-app routing he likes (html‚Üíbrowser, sql‚Üívscode) is just OS file-association and was already consistent ‚Äî untouched.
- **Fix (terminal-scoped, `links.js` + `styles.css` only).** (1) New `logicalLine()` rebuilds the full wrapped logical line by walking `isWrapped` continuations and concatenating untrimmed full-width rows; the provider scans that and maps match offsets back to multi-row `(x,y)` ranges. (2) New `BARE_RE` (whitelisted code/doc/data extensions + lookbehind) folded into a terminal-only `TERM_PATH_RE`; the transcript stays on the strict slash-only `PATH_RE` (prose is English-dense ‚Üí keep it tight). (3) `openPath` now surfaces a transient `#pathlink-flash` toast on backend rejection, naming the file + reason.
- **Verified.** `node --check` clean; `cockpit/test_backend.py` 79/79 (backend untouched); `styles.css` braces balanced (365/365); a throwaway Node fixture confirmed the regex set (long slash path / bare sql / windows drive path MATCH; `2/3/5`, `06/04`, `claude.ai`, `1.5`, `e.g.` correctly IGNORED) and that a 47-char path split at a 40-col boundary reassembles whole spanning row0‚Üírow1. **RUNTIME-UNVERIFIED**: the running cockpit window holds stale JS ‚Äî needs a relaunch to eyeball the live hover/open behavior. Accepted residual documented in code: prose `Word.<whitelisted-ext>` (e.g. `Node.js`) can still match in the terminal ‚Äî rare, and a mis-click just flashes "not found" (the false-negative cure wins the trade).
