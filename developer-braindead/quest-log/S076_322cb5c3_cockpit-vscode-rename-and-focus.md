- **Why VSCode couldn't rename before.** The cockpit's `/rename` is a *keystroke mirror* in [[term.js]] (`_handleData`/`_linebuf`) → browser `localStorage` via [[names.js]]. A VSCode session never touches that terminal and can't reach that localStorage. The only channel connecting an external session to the cockpit is the hook → `switchboard/state-*.json` → `backend.py` → board pipeline. So the rename had to ride *that*. Verified the constraint first (via claude-code-guide): an **unknown** slash command is rejected client-side *before* `UserPromptSubmit` fires — so `/rename` had to become a *real* command first.

- **Feature #1 — `/rename` for any host (silent, zero model turn).** New `brain/.claude/commands/rename.md` makes `/rename` recognized; its body expands to a `<cockpit-rename>$ARGUMENTS</cockpit-rename>` sentinel. New hook [[rename-intercept.py]] on `UserPromptSubmit` (registered *first* in the array in `.claude/settings.json`, ahead of `status-sidecar.py`): matches the sentinel **or** a raw `/rename`, sanitizes the name, writes `switchboard/state-names.json[sid8]=name`, and **exits 2 to block** — the prompt is erased, stderr shows a one-line confirmation to the user only, no model turn. Defensive: any non-rename prompt or parse failure exits 0 (never blocks a real prompt). [[backend.py]] `build_session_model` loads `state-names.json` → new `name` field; [[board.js]] renders `nameFor(sid8) || s.name || actor` (localStorage rename still wins for cockpit terminals; disk name serves VSCode). Added `switchboard/state-names.json` to `.gitignore` (runtime data, like the sibling state files).

- **Feature #2 — click a vscode-host row → open in VSCode.** The `niksis8.claude-focus-0.0.1` extension ([[S037_terminal_switchboard_phase_3_click_to_focus|S037]]/[[S038_vscode_claude_focus_extension|S038]]) is **still installed** and still reads `~/.claude/status/<sid8>.json`'s `claude_pid_chain` (status-sidecar still writes it). New `GET /api/open-vscode?sid8=` in [[backend.py]] fires the extension's URI via `os.startfile(vscode://niksis8.claude-focus/focus?sid8=)` — server-side because WebView2 won't hand a `vscode://` to the OS handler from in-page nav, but ShellExecute does, no prompt. [[main.js]] `selectRow`: `host==="vscode"` → `fetch` the route, then falls through to the read-only peek so the cockpit keeps a selection.
