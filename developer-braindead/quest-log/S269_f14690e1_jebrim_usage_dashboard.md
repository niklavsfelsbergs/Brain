# S269 — 2026-06-18 — Jebrim usage dashboard (tools/jebrim_usage.py)

Built a self-contained, premium HTML/SVG **usage snapshot of how Jebrim is used** — `tools/jebrim_usage.py` → `tools/jebrim-card.html` (regenerable, stdlib-only, reads the Jebrim namespace + `git`). Entered dev-brain via "Lets develop gielinor"; OPEN→UPDATE→CLOSING posted to `comms/active.md`. Long collaborative-iteration session with Niklavs.

- **v0 → v1 (5 polish passes).** First cut had generic GitHub-stats panels (commit heatmap, file-count donut, throughput). Iterated 5× to a premium dark+gold look (Cinzel/Inter, glass cards, gradient bars, sigil, entrance animations, vignette, responsive/print). Fixed two real bugs mid-pass: delegations regex `_[dgp]\d` false-matched sid8 hashes (→ `_[dgp]\d{1,2}_`); peak-ring compared a string to a date so it never drew.
- **Recon-grounded redesign (3 dwarves).** Niklavs: "really read the brain." Fanned out 3 read-only Jebrim dwarves (work-types/outcomes, knowledge/identity, operational/temporal). Findings reframed the whole card: the brain-repo counts measure *bookkeeping by-product*, not work — the real signal is project arcs, cross-repo deliverables, sub-agent fan-out, knowledge growth, recurrence, graduation cadence. Synthesized a value-first proposal; Niklavs picked **value-first + 2 health panels**.
- **v2/v3 rebuild.** Cut heatmap + donut + the misleading raw quest count. New panels: **Project arcs** (per-project sessions + flagship deliverable folded in), **Current focus** (recency), **Job to be done** (recon-derived), **Work rhythm** (quests/week), **Knowledge moat** (cumulative notes+lessons), **Domains × freshness**. Headline restored to the work-volume signals Niklavs wanted (sessions/commits/quests/in-progress).
- **Counting corrections (the load-bearing fixes).** (1) **Sessions = wrap-ups = distinct SNNN** (allocated at close, one per close ritual) → 173/174, cross-checked vs 159 comms `CLOSING`s + ~14 pre-comms early sessions. Old 226 was junk (nameless dwarf sub-quests each counted as a phantom session). (2) **"Quests" was inflated** by 88 sub-agent traces (`_d1_`/`_g1_`); counting **parent quests only** → 187, which reconciles to ~1:1 with 173 sessions ([[S034_guthix_consultation_mode|S034]] alone = one wrap-up emitting 17 quest files). Added a caption making the `sessions → quests + fan-out` relationship legible.
- **UI fixes from live review.** Redesigned project bars from a ragged date-positioned Gantt (SVG) to **aligned magnitude bars** (HTML, left-anchored, color = client/system, span as text, no dots); fixed the **scrolling-background colour drift** (3 radial glows on the scrolling body → one fixed viewport-anchored warm glow, green removed); fixed **chart label clipping** (work-rhythm top headroom; knowledge-moat right margin 56→80px).

Standing decisions Niklavs steered: prefers **computed over curated** (skipped the curated "value delivered €1.9M" panel); job-mix is the one labelled-curated bit (slugs are topic-named, not verb-named, so an auto-count is unreliable).
