# S107 — 2026-05-27 — comms/active.md rotation + read-discipline fix

- Principal (dev-brain, mid-conversation) asked whether sessions read the **full** `comms/active.md` at respawn or just the recent tail, given the channel is append-only and never pruned. Answer: the ritual says read the **tail** (`spellbook/respawn-ritual.md` step 6 "last ~20 entries"; `respawn.md` "Files to read first" marks it `(tail)`) and growth is meant to be bounded by **rotation**, not deletion — but two gaps had made that not pay off in practice.
- **Gap 1 — bare `Read` pulls the wrong end.** The file is append-only with newest at the *bottom*; a `Read` with no offset returns the *oldest* entries (top) and truncates at ~one page, so the live tail is off the end. Surfaced concretely: the dev `comms/active.md` had grown to **976 lines / ~100k tokens** and a default `Read` returned lines 1–205 (May-22 entries) under a truncation warning. To read the tail this session I had to seek to `offset 940`.
- **Gap 2 — rotation trigger lagged the real cost.** The old trigger was "~500 entries," but the dev file was already ~100k tokens at only ~179 entries — token weight, not entry count, is what makes a respawn read expensive. Same bloat lesson [[respawn.md]] already learned (its line-7 token-audit note); comms can't use overwrite-in-place (append-only is load-bearing for concurrent-write safety) so it must lean on rotation, which had never fired.
- **Rotated both channels** (the sanctioned overwrite exception in `_about.md` → Rotation; no live siblings): dev `comms/active.md` 976→**300 lines** (127 oldest entries → [[active-2026-05-27]] archive, kept recent 52); gielinor `comms/active.md` 402→**202 lines** (61 → archive, kept recent 49). 179 and 110 entries respectively preserved across active+archive — nothing deleted, git-reversible.
- **Amended both `comms/_about.md`** (dev + gielinor): rotation now triggers on **token/line weight (~300 lines / ~30k tokens)**, not a fixed entry count, with the 2026-05-27 rotation recorded as the anchoring observation; Read cadence now warns that a bare `Read` returns the *oldest* entries first and to **seek to EOF** (large `offset`, or `grep -n '^\[' … | tail`) for the live tail.
