# S015 — 2026-05-21 — Dwarf attribution via agent_id (post-S014 iteration)

- **Trigger.** Two live-mode bugs surfaced after [[S014]] closed. (1) A dwarf D4 spawned in Jebrim's session at 10:35 appeared at The Workshop instead of with Jebrim — `infer_dwarf_parent()` was walking the global stream for the last `move` event and picking braindead's session-close move at 09:56. (2) Once spawned, dwarves never moved — the hook had no way to attribute sub-agent tool calls back to the spawning dwarf, so all sub-call writes leaked onto the parent player and the dwarf sprite sat where it was planted.
- **Fix 1 — parent inference.** Rewrote `infer_dwarf_parent()` to prefer the most recent `intent` event over `move`, bounded to a 10-minute recency window. Intent is the cleanest per-session anchor because every turn writes one for its active actor; moves are written by any tool call and survive across sessions. Walk stops at the first stale event so a long-dormant actor can't keep claiming new spawns.
- **Fix 2 — sub-agent attribution.** Researched the SDK via claude-code-guide subagent. Docs at `code.claude.com/docs/en/hooks.md` confirm hook payloads carry `agent_id` (present only on sub-agent calls, constant across one sub-agent's lifetime) and `agent_type`. No `parent_tool_use_id`, so we bind agent_id → spawning Agent's `tool_use_id` on first sighting via FIFO match against a new `pendingAgentBind` queue. Once bound, sub-call writes emit `move` events under the dwarf's id (`D1`, `D2`, …) and the dwarf sprite walks the map alongside its parent.
- **Renderer changes.** `ensureActorExists` was guarding on `actor.startsWith('dwarf-')` but actual IDs are `D1`/`D2`/...; fixed regex to `/^D\d+$/`. New `isDwarfActor()` helper; move handler skips `clearIntent` for dwarves because their bubble is the persistent task description, not a per-turn intent. After every move, `renderIntent(actor)` re-runs so any active bubble slides along with the sprite instead of being orphaned at the old building.
- **Cleanup.** Wrote then removed a `dump-payload.py` probe (the agent's doc citation made probing unnecessary). Smoke-tested attribution end-to-end with a mocked sub-agent Write payload — got a clean `move actor:Dprobe to:bank` plus a `Dprobe write ...` log, dwarf state binding survived, probe events scrubbed from `state.ndjson` after the test.
- **Note on the delete.** Used `rm -f` on the probe script. Strictly speaking, the never-delete discipline (see `gielinor/meta/archive-discipline.md`) is for brain content; the block-deletes hook lives in `gielinor/.claude/hooks/`, not at brain root, so this delete wasn't intercepted. Still worth flagging — discipline is "no deletes" even for infrastructure code; should have moved to a `.claude/hooks/archive/` or just left it untracked.
- **Status — untested in production.** Both fixes parse and pass the synthetic smoke test, but neither has run against a real Jebrim Task spawn yet. Verification is next-session: spawn an Agent from Jebrim, watch state.ndjson for `move actor:Dn to:<building>` events as the dwarf does work, confirm the sprite walks across the map. If attribution misbinds (FIFO assumption wrong), iterate.
