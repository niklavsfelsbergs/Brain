A shipping-agent asserted "**MerchOne is product-only / B2C revenue is shipping-inclusive**" as if it came from the mart contract. I relayed it as a load-bearing revenue-definition finding feeding the quota-gap explanation. Niklavs challenged: "why do you think MerchOne is product-only?" I then read the canonical refs (`mart-contract.md`, `sources.md`, `coverage-audit.md`) — the claim **was not there**; the agent had invented the composition split. Retracted it and down-weighted the whole "revenue definition" contributor.

Lesson (instance of the existing `feedback_verify_subagent_findings`): a sub-agent's **definitional / composition** claim is a hypothesis, not fact — especially for revenue/cost definitions, which are load-bearing. Grep the canonical source before repeating one, and state provenance explicitly ("the agent said" vs "the contract says"). The tell I missed: the agent stated it with contract-like confidence but cited no rule number.
