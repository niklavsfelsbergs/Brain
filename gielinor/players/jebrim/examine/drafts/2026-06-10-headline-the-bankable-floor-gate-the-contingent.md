# Headline the bankable floor; gate (and visually downplay) the low-confidence slice

**Observation ([[S191_36b49f0c_eu-tender-annualization-report|S191]], sid8 36b49f0c, EU-tender annualization report).** I first built the annualization report
headlining the *total* saving (€1.34M / 9.2%). Niklavs pushed back: ~61% of it (€806k) rests on the DB Schenker
reroute, which is contingent on an *open* investigation (can those freight parcels actually leave freight?) plus
the unvalidated Maersk-EU/Hermes engines — so leading with the total over-claims. Across several rounds he drove:
(1) reframe to a **two-tier headline** — firm floor €535k leads, contingent gated; (2) the report is **standalone**,
so it must *explain what the contingent lever is* (freight ~€53/parcel → parcel ~€7–14), not just name it — in every
place it surfaces (KPI card, summary, the section); (3) split the **by-destination** table into firm/contingent
columns; (4) the contingent's **yellow** color read as "important" — switch to **muted gray** to downplay it.

**Why it matters.** For a deliverable that headlines a number with a soft, not-yet-bankable component, the honest
default is: **lead with the bankable floor, present the uncertain slice as gated upside, and make the framing
consistent across copy AND visual treatment.** Color is part of the argument — yellow/accent says "look here,"
which fights a "don't bank this yet" message; gray makes it recede. And in a standalone artifact, the reader is cold:
state *what* a lever is and *why it saves* briefly wherever it appears, don't assume project context.

**How to apply.** When a saving/result has a low-confidence sub-component: split it out as its own tier with the
floor leading; explain the lever in-line (standalone-safe); carry the split into the supporting tables so the
asymmetry is visible per-segment; and pick colors that match the emphasis you intend (recede the contingent, don't
highlight it). Related: [[feedback_rerating_savings_guards]] (PAPER vs DEFENSIBLE), the Q1 report's own
DB-Schenker-vs-other saving_split, and the audit's "headline was a Q1 best-case, not a floor" theme.
