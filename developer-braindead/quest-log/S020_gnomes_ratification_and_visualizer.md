## Carried-over observations

- **Env-var sub-agent gating was a multi-hook silent failure mode.** Three hooks shipped over multiple sessions all gated on env vars Claude Code doesn't set. The dwarf hook went undetected for the longest — never tested because no architectural challenge to the dwarf boundary ever ran against a real dwarf doing prohibited writes. Pattern: **architectural guarantees need a live failure test, not just a code review**, because the predicate's-correctness is the load-bearing piece and code review can't catch a wrong predicate when the predicate "reads right." Worth surfacing as a `lorebook/drafts/` entry on the gielinor side at next bankstanding.
- **The claude-code-guide agent was the right tool for the verification.** Direct domain question ("does Claude Code propagate env vars to sub-agents") with a concrete-needed-answer ("yes/no + canonical mechanism if not"). One-shot, returned the precise field names + payload shape needed for the fix. Confirms the dwarf-spawn heuristic from [[D-014]] applies: parallelizable, scoped, domain-specific lookups are exactly where sub-agent invocation pays off.
